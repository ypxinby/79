## 4. 调度与硬件资源

### 4.1 调度

- CPU 时钟：SysConfig 默认 32 MHz。
- SysTick：10 kHz，周期 100 µs。
- 100 µs ISR：电机软件 PWM、云台 STEP、舵机 PWM、超声波时序。
- 5 ms 主循环任务：`Gimbal_Update5ms()`，pending 上限 2。
- 10 ms 主循环任务：视觉双轴 tracker 与 `GimbalTracker_Update(0.010f)`，pending 上限 2。
- 20 ms 主循环任务：`App_Update_20ms()`，使用单个 bool due 标志。
- GPIO GROUP1 中断：编码器 A/B 两轮 ENCA 双边沿。

### 4.2 UART

| 用途 | 外设 | 引脚 | 配置 |
|---|---|---|---|
| 视觉二进制协议 + 调参文本 REPL | UART3 | TX PB2 / RX PB3 | 115200, 8N1, RX IRQ, FIFO 关闭 |

当前没有独立 Debug UART。P1 默认不向视觉 UART 主动发送周期遥测，避免干扰现有视觉设备；只建立非阻塞遥测抽象与快照接口。

### 4.4 当前未使用资源

- SysConfig 未配置硬件 Timer PWM。
- 未配置硬件 Watchdog；P0.5/P1 先建立 watchdog 封装、heartbeat 和故障路径，是否启用硬件 WWDT 需后续确认时钟、复位策略与调试行为。
- 编码器接口存在，但当前无速度/里程消费者。

## 5. 改造前已知问题

1. `TURN_TO_YAW` 底层状态机无内部超时；任务动作级 timeout 能保护普通任务，但固定绕障直接调用底层转角，可能无限旋转。
2. 避障失败只 Stop + `ExternalHold=true`，没有切换 `CAR_STATE_ERROR`，会表现为 RUN 状态永久静默停车。
3. 无主循环 heartbeat、硬件 watchdog 封装和独立软件急停。
4. 20 ms due 使用 bool，积压时无法统计丢周期；5/10 ms pending 会对陈旧输入连续补跑。
5. IMU 与航向控制 dt 固定为 0.02 s，未使用真实控制周期。
6. 单个 10 kHz SysTick 同时承担四类软件时序，且未显式调整优先级。
7. 底盘开环；编码器读取丢方向且无调用者（本轮禁止修改）。
8. 循迹/路口/丢线主要为单帧判定（本轮禁止引入 TrackStateEstimator/LineControl）。
9. 参数无 Flash 持久化（本轮禁止实施）。
10. 视觉 UART 同时承载二进制 RX 和文本调参 TX/RX，不适合默认加入主动周期文本遥测。

## 6. 最小回归测试清单

所有实车测试前先架空驱动轮或使用安全测试台；准备实体断电手段。

### R0 编译与启动

- [ ] CCS Debug 全量编译无 error；记录 warning。
- [ ] 上电后 OLED、按键、IMU 初始化正常，电机保持停止。
- [ ] UART3 视觉协议能持续接收，CRC/overflow 计数无异常增长。

### R1 基础循迹任务

- [ ] 开发 Profile 选择 `TEST-BASIC`，执行 SEEK → FOLLOW。
- [ ] 能找到线并稳定循迹；速度、KP/KD 与动作数组保持基线值。
- [ ] STOP/完成/失败时电机输出为 0。

### R2 TURN_TO_YAW

- [ ] 运行 `TEST-YH` 或等价诊断动作。
- [ ] 正常完成相对转角，成功条件仍为误差 ≤5° 且稳定 100 ms。
- [ ] 人为反转方向宏/阻挡转动/制造不可达目标，验证底层超时停车并进入可识别故障。

### R3 DRIVE_HEADING_TIME

- [ ] `TEST-YH` 中按当前 yaw 保持航向直行 700 ms。
- [ ] IMU 未就绪时应停止并报告错误。
- [ ] 正常持续时间完成后电机停止并推进下一动作。

### R4 STOP_ONLY 避障

- [ ] 开发 Profile 选择 `TEST-STOP`。
- [ ] 连续确认障碍后 SafetyHold 生效、任务时间冻结。
- [ ] 清除障碍后解除 Hold，恢复原 FOLLOW，不切错 action_index。

### R5 FIXED_BYPASS 避障

- [ ] 开发 Profile 选择 `TEST-OBS-F`。
- [ ] 验证等待 → 转出 → 航向直行 → 转回 → 回线稳定 → 重下发原 FOLLOW。
- [ ] 正常完成时 `MotionAction_ReapplyControllerTarget()` 链路不回退。
- [ ] 人为制造转角/回线失败，验证保持 Hold、任务冻结并进入 ERROR，而不是 RUN 静默停车。

### R6 暂停、继续、取消、复位

- [ ] RUNNING 时 K2 暂停，电机停止。
- [ ] PAUSED 时 K2 继续；记录当前实现为动作重启的已知行为。
- [ ] K3 短按取消后进入 READY，电机停止。
- [ ] K3 长按复位后任务和底层运行态回到 READY。

### R7 软件急停

- [ ] 通过独立 EmergencyStop 接口/调试命令触发。
- [ ] 触发后立即 `Motor_Stop`，任务、避障、Resume 均不能重新驱动。
- [ ] 普通 Pause/Resume 不能解除急停。
- [ ] 仅明确的 EmergencyStop reset 操作可以解除，解除后仍保持 READY，不自动续跑。

### R8 heartbeat/watchdog

- [ ] 正常主循环完成关键 20 ms 控制周期后 heartbeat 更新。
- [ ] 模拟主循环不再完成控制周期，heartbeat 超时路径置故障并保持停车。
- [ ] 若硬件 watchdog 未启用，确认状态明确显示 `HW watchdog disabled`，不得宣称已验证硬件复位。

### R9 云台与视觉并行

- [ ] 视觉目标帧持续输入时 YAW/PITCH tracker 正常。
- [ ] 5 ms 云台更新、10 ms tracker、20 ms 底盘并行运行，无明显丢步/卡死。
- [ ] 记录 5/10/20 ms missed/drop/overrun 与 UART overflow。
