# 步进电机二维云台运控逻辑修订版

## 1. 运控目标

二维云台采用两路步进电机驱动：

* `PITCH`：俯仰轴；
* `YAW`：水平轴。

两轴均按照可完成 360°旋转设计。运控系统应同时支持：

1. 双轴独立速度控制；
2. 相对角度运动；
3. 绝对单圈角度控制；
4. 连续累计角度控制；
5. 最短路径转动；
6. K230 视觉目标跟踪；
7. 目标丢失和通信超时保护；
8. 步进脉冲计数位置估算。

需要注意：步进电机能够旋转 360°，不代表软件中只需要保存 `0~360°`。如果云台连续旋转多圈，仅保存单圈角度会丢失累计圈数，因此系统必须同时维护单圈角度和连续角度。

---

## 2. 角度与位置模型

### 2.1 每轴核心状态

```c
typedef struct {
    int64_t estimated_steps;

    float continuous_angle_deg;
    float wrapped_angle_deg;
    int32_t turn_count;

    float target_continuous_angle_deg;
    float target_rpm;
    float commanded_rpm;

    int8_t direction;

    bool enabled;
    bool step_running;
    bool position_valid;
} gimbal_axis_state_t;
```

字段含义如下。

| 字段                            | 含义                          |
| ----------------------------- | --------------------------- |
| `estimated_steps`             | 根据有效 STEP 脉冲累计的软件位置，支持正负和多圈 |
| `continuous_angle_deg`        | 连续累计角度，可以大于 360°或小于 0°      |
| `wrapped_angle_deg`           | 单圈角度，范围为 `0~360°`           |
| `turn_count`                  | 相对软件零点累计旋转圈数                |
| `target_continuous_angle_deg` | 连续坐标系中的目标角度                 |
| `target_rpm`                  | 控制器计算出的目标转速                 |
| `commanded_rpm`               | 经过加减速限制后实际下发的转速             |
| `direction`                   | 当前运动方向，`+1/-1`              |
| `position_valid`              | 软件零点和位置估算是否可信               |

---

### 2.2 连续角度

连续角度不进行取模：

```text
0°
90°
360°
450°
720°
-90°
```

均为合法值。

连续角度根据累计步数计算：

```c
continuous_angle_deg =
    estimated_steps / steps_per_degree;
```

其中：

```c
steps_per_degree =
    motor_full_steps_per_rev
    * microstep
    * gear_ratio
    / 360.0f;
```

不能固定写死为 `0.1125°/step`，除非已经确认：

```text
电机步距角 = 1.8°
细分 = 16
传动比 = 1:1
```

---

### 2.3 单圈角度

单圈角度用于显示、绝对角度命令和最短路径计算。

```c
float wrap_angle_360(float angle)
{
    float result = fmodf(angle, 360.0f);

    if (result < 0.0f) {
        result += 360.0f;
    }

    return result;
}
```

更新方式：

```c
axis->wrapped_angle_deg =
    wrap_angle_360(axis->continuous_angle_deg);
```

结果范围为：

```text
0.0 <= wrapped_angle_deg < 360.0
```

---

### 2.4 圈数计算

```c
axis->turn_count =
    (int32_t)floorf(axis->continuous_angle_deg / 360.0f);
```

例如：

|   连续角度 |   单圈角度 |   圈数 |
| -----: | -----: | ---: |
|  `90°` |  `90°` |  `0` |
| `450°` |  `90°` |  `1` |
| `810°` |  `90°` |  `2` |
| `-90°` | `270°` | `-1` |

## 3. STEP 计数规则

业务层只统计有效 STEP 脉冲，不统计 GPIO 翻转次数。

推荐定义：

```text
每产生一个有效 STEP 上升沿：
    estimated_steps += direction
```

伪代码：

```c
void gimbal_on_step_rising_edge(gimbal_axis_t axis_id)
{
    gimbal_axis_state_t *axis = get_axis(axis_id);

    if (!axis->step_running) {
        return;
    }

    axis->estimated_steps += axis->direction;

    axis->continuous_angle_deg =
        axis->estimated_steps / axis->steps_per_degree;

    axis->wrapped_angle_deg =
        wrap_angle_360(axis->continuous_angle_deg);

    axis->turn_count =
        calculate_turn_count(axis->continuous_angle_deg);
}
```

如果底层定时器采用 Toggle 输出，两次翻转产生一个完整脉冲，这属于底层驱动实现细节。底层只能在确认产生了有效上升沿时通知业务层计一步。

位置运动完成后不能再次执行：

```c
current_angle += move_angle;
```

否则会造成角度重复累计。

## 4. 三种位置命令

### 4.1 相对角度运动

接口：

```c
gimbal_move_relative(axis, delta_angle_deg);
```

处理：

```c
axis->target_continuous_angle_deg += delta_angle_deg;
```

例如当前连续角度为 `450°`：

```text
相对 +30° -> 目标 480°
相对 -90° -> 目标 360°
```

相对角度命令不进行 `0~360°` 取模。

---

### 4.2 连续绝对角度运动

接口：

```c
gimbal_set_continuous_target(axis, target_angle_deg);
```

示例：

```text
目标 720°：运动到相对零点正向两圈
目标 -360°：运动到相对零点反向一圈
目标 1080°：运动到相对零点正向三圈
```

此模式不使用最短路径，直接在连续角度坐标系中计算：

```c
error =
    target_continuous_angle_deg
    - continuous_angle_deg;
```

---

### 4.3 单圈绝对角度最短路径运动

接口：

```c
gimbal_set_wrapped_target(axis, target_angle_0_360);
```

先计算当前单圈角度：

```c
current_wrapped =
    wrap_angle_360(axis->continuous_angle_deg);
```

计算最短路径误差：

```c
float shortest_angle_error(float target, float current)
{
    float error = target - current;

    while (error > 180.0f) {
        error -= 360.0f;
    }

    while (error < -180.0f) {
        error += 360.0f;
    }

    return error;
}
```

再把单圈目标映射到距离当前位置最近的连续目标：

```c
float error =
    shortest_angle_error(target_wrapped,
                         axis->wrapped_angle_deg);

axis->target_continuous_angle_deg =
    axis->continuous_angle_deg + error;
```

例如：

```text
当前位置：350°
目标角度：10°
最短误差：+20°
连续目标：当前位置连续角度 +20°
```

又例如：

```text
当前位置：10°
目标角度：350°
最短误差：-20°
```

这样可以保留 360°云台的最短路径优势，同时不会丢失累计圈数。

当误差恰好为 `180°` 时，应通过配置决定固定选择正向还是负向，避免方向随机变化。

## 5. 位置伺服控制

### 5.1 控制误差

位置控制统一在连续角度坐标系中执行：

```c
error_deg =
    target_continuous_angle_deg
    - continuous_angle_deg;
```

单圈绝对目标在进入控制器之前，已经映射成最近的连续目标，因此控制器内部不需要再次执行角度取模。

---

### 5.2 PD 控制器

第一版建议使用 PD，不启用积分项：

```c
derivative =
    (error_deg - last_error_deg) / dt;

target_rpm =
    kp * error_deg
    + kd * derivative;
```

然后限幅：

```c
target_rpm =
    clamp(target_rpm,
          -max_rpm,
          max_rpm);
```

推荐控制周期：

```text
dt = 0.005s
控制周期 = 5ms
```

原 STM32 代码的微分项没有除以 `dt`，迁移后如果改成标准微分形式，`Kd` 必须重新调节，不能直接继续使用原来的 `0.5`。

---

### 5.3 死区

```c
if (fabsf(error_deg) <= position_deadband_deg) {
    target_rpm = 0.0f;
}
```

初始建议：

```text
position_deadband_deg = 0.2°~0.5°
```

实际死区应根据细分、机械抖动和最低稳定转速调整。

---

### 5.4 加速度限制

应使用与时间有关的加速度限制，而不是固定“每次最多变化 3 RPM”。

```c
float max_delta_rpm =
    acceleration_limit_rpm_per_s * dt;

float rpm_diff =
    target_rpm - commanded_rpm;

rpm_diff =
    clamp(rpm_diff,
          -max_delta_rpm,
          max_delta_rpm);

commanded_rpm += rpm_diff;
```

例如：

```text
acceleration_limit = 600 RPM/s
dt = 0.005s
每周期最大变化 = 3 RPM
```

这样即使以后控制周期修改，物理加速度限制仍保持一致。

---

### 5.5 换向保护

当目标 RPM 和当前 RPM 方向相反时，必须先减速到零，再改变 DIR：

```text
正向运行
    ↓
逐渐减速
    ↓
停止 STEP
    ↓
设置新 DIR
    ↓
等待 DIR 建立时间
    ↓
从零开始反向加速
```

伪代码：

```c
if (sign(target_rpm) != sign(commanded_rpm)
    && fabsf(commanded_rpm) > stop_rpm_threshold) {

    target_rpm = 0.0f;
}
```

只有当：

```c
fabsf(commanded_rpm) <= stop_rpm_threshold
```

时，才允许修改方向。

---

## 6. 速度到 STEP 频率转换

RPM 转 STEP 频率：

```c
step_frequency_hz =
    fabsf(rpm)
    * steps_per_revolution
    / 60.0f;
```

其中：

```c
steps_per_revolution =
    motor_full_steps_per_rev
    * microstep
    * gear_ratio;
```

定时器半周期：

```c
half_period_ticks =
    timer_frequency_hz
    / (2.0f * step_frequency_hz);
```

需要同时限制：

```text
最小稳定 STEP 频率
最大安全 STEP 频率
最小脉冲高电平时间
最小脉冲低电平时间
定时器最大计数范围
```

当目标转速过低时，不应把频率无限降低，而应停止 STEP：

```c
if (fabsf(commanded_rpm) < minimum_effective_rpm) {
    stepper_stop(axis);
}
```

## 7. K230 视觉运控逻辑

### 7.1 K230 与 MSPM0 职责

K230 负责：

```text
摄像头采集
模型推理
目标检测
多目标筛选
目标中心计算
置信度判断
目标丢失判断
发送视觉数据
```

MSPM0G3507 负责：

```text
协议解析
视觉误差计算
视觉外环控制
连续目标角更新
位置伺服
STEP/DIR/EN 输出
超时保护
```

K230 不直接产生 STEP，也不直接控制 D36A。

---

### 7.2 坐标映射

正式使用以下轴命名：

```text
图像 X 误差 -> YAW 水平轴
图像 Y 误差 -> PITCH 俯仰轴
```

根据帧内宽高动态计算中心：

```c
center_x = frame_width * 0.5f;
center_y = frame_height * 0.5f;

error_x_px = target_x - center_x;
error_y_px = target_y - center_y;
```

归一化误差：

```c
error_yaw =
    2.0f * error_x_px / frame_width;

error_pitch =
    2.0f * error_y_px / frame_height;
```

归一化误差范围约为：

```text
-1.0 ~ +1.0
```

方向最终通过配置控制：

```c
error_yaw *= YAW_VISION_DIRECTION;
error_pitch *= PITCH_VISION_DIRECTION;
```

其中方向系数为：

```text
+1 或 -1
```

---

### 7.3 视觉死区

使用归一化死区，不再固定为 `20px`：

```c
if (fabsf(error_yaw) < yaw_deadzone) {
    error_yaw = 0.0f;
}

if (fabsf(error_pitch) < pitch_deadzone) {
    error_pitch = 0.0f;
}
```

初始建议：

```text
yaw_deadzone = 0.04~0.08
pitch_deadzone = 0.04~0.08
```

例如死区为 `0.05`，代表图像中心左右或上下约 2.5%的范围内不调整。

---

### 7.4 视觉外环输出角速度

K230 视觉误差不应在每次收到帧时直接转换成固定角度增量。更稳定的方式是转换为目标角速度：

```c
vision_yaw_velocity_deg_s =
    vision_kp_yaw * error_yaw;

vision_pitch_velocity_deg_s =
    vision_kp_pitch * error_pitch;
```

可以加入非线性增益：

```c
if (fabsf(error) > fast_threshold) {
    velocity = fast_gain * error;
} else {
    velocity = slow_gain * error;
}
```

再执行限幅：

```c
velocity =
    clamp(velocity,
          -max_vision_velocity_deg_s,
          max_vision_velocity_deg_s);
```

每个 5ms 控制周期根据目标角速度更新连续目标角：

```c
target_continuous_angle_deg +=
    vision_velocity_deg_s * dt;
```

这种方式允许在 K230 两帧之间继续平滑运动，并且不会因为同一帧被重复使用而重复叠加固定角度。

---

### 7.5 K230 数据时间戳

每次收到合法视觉帧后记录：

```c
latest_target.received_time_ms =
    get_time_ms();
```

5ms 控制任务计算：

```c
vision_age_ms =
    current_time_ms
    - latest_target.received_time_ms;
```

状态处理：

| 状态    | 条件                       | 运控行为                  |
| ----- | ------------------------ | --------------------- |
| 有效目标  | `valid=1` 且 `age<=100ms` | 根据视觉误差产生目标角速度         |
| 正常无目标 | `valid=0` 且通信正常          | 目标角速度设为 0，保持最后位置      |
| 短暂延迟  | `100ms<age<=300ms`       | 目标角速度逐渐衰减到 0          |
| 通信失效  | `age>300ms`              | 停止视觉更新，停止 STEP，保持电机使能 |
| 协议错误  | CRC、版本或字段非法              | 丢弃当前帧，不更新目标           |

---

### 7.6 每个视觉帧只更新一次观测状态

使用序列号判断新帧：

```c
if (frame.sequence != last_sequence) {
    latest_target = frame;
    last_sequence = frame.sequence;
    update_vision_observation();
}
```

但目标角积分仍由固定 5ms 控制周期执行：

```c
target_angle += vision_velocity * 0.005f;
```

这样能够同时做到：

1. 不重复解析同一帧；
2. 不重复计算视觉微分项；
3. K230 帧率变化时，运动速度基本保持一致；
4. MSPM0 的位置伺服始终保持固定周期。

## 8. K230 视觉外环与位置内环

整体控制链路为：

```text
K230 目标坐标
    ↓
归一化视觉误差
    ↓
视觉外环计算目标角速度
    ↓
连续目标角积分
    ↓
目标角度与软件估算角度误差
    ↓
位置 PD 输出目标 RPM
    ↓
加减速限制和换向保护
    ↓
STEP 频率
```

需要注意，位置反馈来自 STEP 计数，因此属于软件位置估算，不是带编码器的真实电机闭环。

K230 的图像反馈构成系统外环，可以纠正目标在画面中的位置，但不能检测电机内部是否发生了丢步。

## 9. 360°旋转模式配置

需要明确区分：

### 9.1 单圈可旋转

机械结构允许从任意角度运动到 `0~360°` 中的任意位置，但线缆不允许无限绕圈。

此时保留累计圈数，并增加最大圈数保护：

```c
#define YAW_MIN_TURNS  (-2)
#define YAW_MAX_TURNS  (+2)
```

当目标连续角度超过允许圈数时，禁止继续向该方向运动。

---

### 9.2 无限连续旋转

如果云台装有导电滑环，机械和线缆均允许无限旋转，可以关闭圈数限制：

```c
#define YAW_TURN_LIMIT_ENABLED  0
```

此时：

```text
continuous_angle_deg
estimated_steps
turn_count
```

仍然需要保留，只是不进行最大圈数限制。

---

## 10. 上电零点

即使云台能够 360°旋转，步进电机本身也不能提供绝对角度，因此断电后累计步数会丢失。

上电后可以选择三种策略。

### 策略一：软件当前位置作为零点

```text
上电时当前位置定义为 0°
position_valid = true
```

优点是简单，缺点是每次上电的物理零点不同。

### 策略二：零点传感器回零

```text
低速寻找零点传感器
触发后停止
将 estimated_steps 设为 0
position_valid = true
```

这是正式系统最推荐的方式。

### 策略三：只进行视觉相对跟踪

如果不关心云台绝对物理角度，可以上电后把当前位置设为零点，K230 只通过相对视觉误差改变目标角度。

第一版视觉跟踪可以采用此策略。

## 11. 模式管理

```c
typedef enum {
    GIMBAL_MODE_DISABLED,
    GIMBAL_MODE_SPEED,
    GIMBAL_MODE_RELATIVE_POSITION,
    GIMBAL_MODE_WRAPPED_POSITION,
    GIMBAL_MODE_CONTINUOUS_POSITION,
    GIMBAL_MODE_VISION_TRACKING,
    GIMBAL_MODE_FAULT
} gimbal_mode_t;
```

模式切换流程：

```text
停止两轴 STEP
等待输出完全停止
清除目标 RPM
清除平滑 RPM
清除 PD 历史误差
将目标连续角设置为当前连续角
清除旧视觉速度
清除未处理视觉帧
切换模式
```

禁止在未清除旧目标的情况下直接从位置测试模式进入视觉模式。

## 12. 5ms 运控任务

```c
void gimbal_control_update_5ms(void)
{
    update_communication_status();
    update_fault_status();

    if (gimbal_has_fault()) {
        stop_all_step_outputs();
        return;
    }

    switch (gimbal_mode) {
    case GIMBAL_MODE_VISION_TRACKING:
        vision_outer_loop_update(0.005f);
        break;

    case GIMBAL_MODE_SPEED:
        speed_mode_update();
        break;

    case GIMBAL_MODE_RELATIVE_POSITION:
    case GIMBAL_MODE_WRAPPED_POSITION:
    case GIMBAL_MODE_CONTINUOUS_POSITION:
        break;

    default:
        stop_all_step_outputs();
        return;
    }

    position_servo_update(PITCH_AXIS, 0.005f);
    position_servo_update(YAW_AXIS, 0.005f);

    apply_acceleration_limit(PITCH_AXIS, 0.005f);
    apply_acceleration_limit(YAW_AXIS, 0.005f);

    update_stepper_output(PITCH_AXIS);
    update_stepper_output(YAW_AXIS);
}
```

高频 STEP 中断中只能执行：

```text
产生 STEP 边沿
更新下一次比较值
更新有效步数
检查定步数运动是否完成
```

STEP 中断中禁止执行：

```text
浮点 PID
UART 打印
K230 协议解析
OLED 刷新
复杂状态判断
```

## 13. 第一版建议参数

以下参数仅作为低速初始值，必须实机调整：

```text
控制周期：5ms

位置死区：0.3°
最大位置伺服转速：30RPM
最低有效转速：0.5~1RPM
加速度限制：200~400RPM/s

视觉归一化死区：0.05
视觉最大角速度：20~40°/s
视觉数据有效期：100ms
通信失效时间：300ms

K230 UART：115200bps，8N1
```

第一次调试应降低：

```text
最大 RPM
最大 STEP 频率
视觉最大角速度
加速度
```

确认方向、计步和最短路径逻辑正确后再逐步提高。

## 14. 实施顺序

重新实现时按照以下顺序进行：

```text
1. 实现单轴 STEP、DIR、EN
2. 验证有效上升沿计步
3. 实现 signed int64 累计步数
4. 实现连续角度、单圈角度和圈数
5. 实现相对角度运动
6. 实现连续绝对角度运动
7. 实现单圈目标最短路径运动
8. 实现双轴独立速度
9. 实现位置 PD 和加速度限制
10. 实现模式切换
11. 用电脑模拟 K230 协议
12. 实现归一化视觉误差
13. 实现视觉目标角速度外环
14. 实现单轴视觉跟踪
15. 实现双轴视觉跟踪
16. 增加通信超时和故障处理
17. 最后连接 K230 实机
```

第一阶段不要同时接入 K230、PID、OLED 和全部小车功能。应先验证单轴连续旋转、累计步数和 `350°→10°` 最短路径是否正确。
