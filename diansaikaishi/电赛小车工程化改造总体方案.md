# 电赛小车工程化改造总体方案

> 平台:TI MSPM0G3507(Cortex-M0+)/ CCS 工程。
> 本方案基于对当前工程源码的**逐文件只读审计**产出,区分"文档声称状态"与"代码真实状态",不臆断硬件,未确认项统一列为**待确认项**。
> 本方案只做设计与迁移规划,**不含大规模代码补丁**;每一阶段可独立提交、独立验收、可回退。
> 生成日期:2026-07-20。

---

## 0. 阅读指引与审计口径

审计覆盖全部应用层与驱动层 `.c/.h`(约 12.5k 行)。每条结论标注状态之一:

- 🟢 **已实现且稳定**
- 🟡 **已实现但存在风险**
- 🔵 **接口存在但未真正使用**
- ⚪ **文档提到但代码未实现**
- 🟣 **建议新增**

核心事实(贯穿全篇,先行说明):

1. **全工程不使用任何硬件定时器 / 硬件 PWM 外设**。电机 PWM、转向舵机 PWM、云台双轴步进 STEP、超声波时序,全部在**单个 10 kHz `SysTick_Handler`** 里用 GPIO 位翻转软件生成(`empty.c:106-146`)。
2. **底盘完全开环**。编码器中断在采集脉冲,但 `Encoder_GetAndClearPulseDeltas()` **全工程无任何调用者**,且读出时 `abs()` 丢弃方向(`encoder.c:49-61`);无 PPR/轮径定义;无速度环、无里程。行驶类动作全部用**时间**判定完成。
3. **架构分层是本工程最扎实的部分**:`Motor_SetSpeed` 全工程唯一调用点在 `car_controller.c:106`,任务/避障/UI 层均不直接碰 PWM;`SafetyHold` 是真正的"禁止最终输出"而非"目标置零";解除 Hold 有防冲出复位。这些必须在改造中**严格保留**。

## 1. 当前工程事实审计

### 1.1 调度与实时性 🟡

| 时基 | 载体 | 内容 |
|---|---|---|
| 100 µs(10 kHz) | `SysTick_Handler`(ISR) | `Motor_PwmTick100us` / `Gimbal_Tick100us` / `Servo_Tick100us` / `Ultrasonic_Tick100us`(`empty.c:113-116`) |
| 5 ms(200 Hz) | 主循环消费 pending | `Gimbal_Update5ms`(云台速度斜坡/STEP 半周期) |
| 10 ms(100 Hz) | 主循环消费 pending | 视觉追踪 `GimbalVision*Tracker_Update10ms` + `GimbalTracker_Update` |
| 20 ms(50 Hz) | 主循环 `g_appUpdateDue` | `App_Update_20ms`(小车任务/巡线/避障/按键/OLED) |
| 事件 | GROUP1 GPIO 中断 | `Encoder_HandleGpioInterrupt`(`empty.c:148-157`) |

- `App_Update_20ms` 调用顺序(`app.c:87-134`):`Imu_Update(0.02f)` → `App_UpdateHeadingObserver` → `Ultrasonic_Update` → `ObstacleMonitor_Update` → 舵机居中 → `Key/Menu` → `MissionManager_Update` → `ObstacleSafety_Update` → `ObstacleAvoidance_Update` → **`CarController_Update_20ms`(唯一真正写电机)** → 调试镜像 → `OledUi_Update`。
- **最重中断负担**:10 kHz SysTick 串行执行 4 个 tick,是全系统实时性瓶颈。
- **潜在阻塞/抖动点**:`SysTick_Config()` 默认把 SysTick 设为**最低优先级**,会被视觉 UART RX 中断与编码器 GPIO 中断抢占,高速步进/高频编码器脉冲下可造成 STEP/PWM 脉宽抖动、丢步。
- **软件解耦良好**:云台链(视觉→追踪→5ms 云台)与底盘链(20ms)在主循环里各自 pending 计数 catch-up,互不调用;但二者**共用同一 10 kHz ISR 的 CPU 时间**,物理时序未完全解耦。

### 1.2 电机与编码器链路 🟡🔵

- **量纲**:`Motor_SetSpeed(int16 A,int16 B)`,抽象占空比 **−1000..+1000**(`motor.h:6` `MOTOR_MAX_DUTY`),非物理速度。典型下发 base=160、turn=130,即约 10–16% 占空。
- **方向定义**:Motor A=左轮,Motor B=右轮(`set_output_speed→Motor_SetSpeed(left,right)`,`car_controller.c:106`);>0 时 xIN1=1/xIN2=0;=0 时方向脚全清 = **滑行(非刹车)**。反向宏 `MOTOR_A/B_INVERT_DIRECTION` 默认 0。
- **PWM**:软件 PWM,`g_pwmPhase` 0..99 @100 µs → **载频仅 100 Hz**(`motor.c:105-123`);|duty| 映射 0..100 档,校正分辨率约 10 单位/档。
- **编码器**:AB 两相接线,但**仅 ENCA 双边沿开中断(×2 解码,非完整 ×4)**;方向在 ISR 里 `(phaseA==phaseB)?+1:−1` 累加为带符号 int32(`encoder.c:37-43`);但 `Encoder_GetAndClearPulseDeltas` 返回 `abs()` → **读出丢方向**,且**无人调用**;临界区读清一致(无竞争/无丢脉冲)。
- **PPR/轮径/齿比**:代码中**完全缺失**。
- **PID 模块**:`pid.c` 完整(含抗积分饱和、输出限幅、Reset),但 `PID_Init/Update/Reset` **无任何调用者** 🔵。

### 1.3 循迹链路 🟡

- 7 路数字量灰度(非 ADC),权重 `{−300,−200,−100,0,100,200,300}`;`TRACK_BLACK_LEVEL` 编译期决定极性(`track_sensor.c:4-15`)。
- 误差 = `Σweight / count`(**除以触发个数而非权重和**),全丢线返回 0(`track_sensor.c:63-80`)——丢线瞬间误差被拉回 0。
- 直角判定 `TrackSensor_DetectTurn` 为**纯单帧组合逻辑**,无时间防抖/候选-确认/置信度;**无独立十字(CROSS)识别**分支 ⚪;丢线 `raw==0` 单帧即切 `LOST_RECOVER`,无消抖。
- 循迹 PD 内联在 `handle_follow_line`(`car_controller.c:292-317`):`correction=(kp*error+kd*(error−last_error))/scale`,clamp `±max_correction`,再 `left=base+corr / right=base−corr`,末端对称 clamp `±1000`。**开环差速,非目标轮速**。
- `last_valid_error`、`lost_line_threshold` 均**只写不读/仅定义** 🔵。
- **参数强耦合**:等效增益=kp/scale;base 与 max_correction 决定内轮能否到 0/反向(默认下内轮最低为 0,**永不反向**,大弯受限);高 base + 高 max_correction 时 `base+corr>1000` 被对称 clamp 破坏差速对称。

### 1.4 IMU 与航向链路 🟡

- 单文件 `imu.c` 混合软件 I2C / MPU6050 寄存器 / 死区滤波 / yaw 积分,**gyro-Z 单轴**。
- 零偏:开机静止 `Imu_CalibrateGyroBias(200)`(弃前 50 帧),有效样本 <90% 判失败 🟢;**无运行期温漂再标定**。
- **dt 固定 0.02**(`Imu_Update(0.02f)` 硬编码),非真实 dt;某帧 I2C 读失败则该帧不积分(时间照走、角度不加)→ 转角累计缺口。
- **yaw 不归一化**(`yaw_deg` 无限累加),环绕处理放在误差侧 `normalize(target−current)`;长时间运行有 float 精度隐患。
- **航向微分用实测 `−gyro_z`**(`heading_control.c:86`),抗噪好,是设计亮点 🟢;输出为差速占空比叠加,非目标轮速。
- IMU 未就绪:巡线内航向**静默降级**(合理);`TURN_TO_YAW`/`DRIVE_HEADING` **非静默,直接停车+ERROR** 🟢。
- **无尖峰/离群点剔除**,仅 1.5 dps 死区。
- `TURN_TO_YAW`:两级 bang-bang(12°/5°),稳定 100 ms 判完成;**无超时保护 → 标度或方向错误时可能无限原地旋转**(对比 90° 转弯有 `turn_max_ms`)。

### 1.5 任务、避障与安全链路 🟢🟡⚪

- 任务=**静态动作数组 + 注册表 + 宏 DSL**(`mission_library.*`),9 种动作(`motion_types.h:8-18`),公共 `timeout_ms/max_retries`;结果码 IDLE/RUNNING/SUCCESS/FAILED/TIMEOUT/CANCELLED 🟢;有静态校验 `MissionLibrary_Validate` 🟢。**纯线性序列,无运行期分支/跳转**。
- 进度归 `MissionManager` 独占;`external_hold` 为真时整体 return,避障期间任务/动作计时冻结 🟢。
- **单一电机输出点、SafetyHold 真正禁输出、解除 Hold 防冲出复位、无越层直出 PWM** 🟢(工程最高质量部分)。
- **风险**:
  - 避障失败 → `avoid_fail` 只置 Hold+Stop,**不升 CAR_STATE_ERROR** → 永久静默 Hold,OLED 显示 RUN 却不动,须人工复位(`obstacle_avoidance.c:62-69` + `obstacle_safety.c:52-54`)🟡。
  - `ObstacleScanner_Update_20ms` **主循环从未调用** → 左右扫描选向能力形同虚设 🔵。
  - Pause/Resume 实为**当前动作重跑**(重采基准+重计时),对 `TURN_TO_YAW`/`DRIVE_HEADING_TIME` 会造成偏差 🟡。
  - **无统一 SafetySupervisor/FaultManager** ⚪;优先级链仅隐式存在于 `ObstacleSafety_Update` 判断顺序;无独立软件急停通道(只能走 Menu→Pause)。
  - **无编码器失效/堵转检测、无看门狗** ⚪(最大安全短板;主环卡死则 PWM ISR 仍按最后档位持续驱动)。

### 1.6 参数与调试能力 🟡⚪

- 参数分散**三处** RAM 结构:`AppConfig`(底盘/巡线/舵机/避障,有默认值+范围钳位,**无单位/版本**)、`VisionPitchTuningParams`、`VisionYawTuningParams`(定点缩放,有 revision,无版本/单位);部分增益仍是**编译期宏**。
- **无任何 Flash 持久化** ⚪:`SAVE` 命令返回 `ERR SAVE_UNSUPPORTED`,`GET PARAM` 固定 `PERSIST=0`;上电全回默认值。
- 调试:**无独立 Debug UART、无 printf**;调试=**复用视觉 UART 的文本 REPL** `vision_tuning_console`(请求-应答,非周期遥测帧)🔵;OLED 诊断页丰富(计数 3 位截断);ISR 内只做环缓冲入队 🟢。
- 视觉链:40 字节定长帧 + CRC16 校验、512B 环形缓冲非阻塞 RX、字节级重同步、双轴追踪状态机 🟢(工程较完善部分)。

### 1.7 待确认项(硬件,禁止臆断)

1. 编码器每相 **PPR、轮径、齿比、轮距**——代码完全缺失,做里程/速度前必须实测补齐。
2. 电机接线正方向(A/B 的 `+` 是否整车前进)、左右轮物理对应、陀螺极性符号。
3. TB6612 PWMA/PWMB 与 AIN/BIN 实际引脚映射(代码假定两 PWM 同在 B 端口)。
4. 7 路灰度实际接线极性与 `TRACK_BLACK_LEVEL` 是否一致;S4 单独位于 A 口有无电气差异。
5. MPU6050 AD0 地址(固定 0x68)与软件 I2C 引脚是否与实车一致。
6. 100 Hz 软件 PWM 在目标电机上的实际线性度/啸叫/低速死区。
7. HC-SR04 ECHO 5V 电平是否已分压后进 MCU。

## 2. 当前主要问题与根因排序

按"对比赛稳定性 / 安全性影响"从高到低:

| # | 问题 | 根因 | 影响 | 状态 |
|---|---|---|---|---|
| P-1 | `TURN_TO_YAW` 无超时保护 | 状态机缺 `turn_elapsed_ms`/超时分支 | 标度或方向配错→无限原地旋转,现场失控 | 🟡 |
| P-2 | 避障失败进入永久静默 Hold | `avoid_fail` 不升 ERROR、不自恢复 | 车"假死"须人工复位,丢失比赛时间 | 🟡 |
| P-3 | 底盘完全开环、无速度/里程反馈 | 编码器数据无人消费、无 PPR/轮径 | 电压/负载/地面变化→直行距离与直线度不可复现 | 🔵⚪ |
| P-4 | 无看门狗/主环存活保护 | 无 WWDT、ISR 不检查主环存活 | 主环卡死则电机按最后档位持续跑 | ⚪ |
| P-5 | 循迹全单帧、无防抖/置信度/十字识别 | 状态估计层缺失 | 高速下单帧抖动→误转弯/误丢线 | 🟡 |
| P-6 | SysTick 最低优先级、全软件位翻转 | 无硬件 PWM、无优先级配置 | 步进/PWM 被 UART/编码器中断抢占抖动 | 🟡 |
| P-7 | IMU 固定 dt、yaw 不归一化、无尖峰剔除 | 估计层与积分耦合 | 长时/丢帧下航向漂移、尖峰直接进积分 | 🟡 |
| P-8 | 无统一 Safety/Fault 仲裁、无软件急停通道 | 安全逻辑分散在各模块 | 故障响应不一致、无一键急停 | ⚪ |
| P-9 | 参数无版本/单位、无 Flash 持久化 | 无持久化层 | 每次上电回默认,现场反复调参 | ⚪ |
| P-10 | Pause/Resume 重跑而非续跑 | Resume 时 MotionAction 重采基准 | 暂停恢复后转角/直行偏差 | 🟡 |

## 3. 改造目标与非目标

**目标**

- 稳定巡线(提速、光照变化、短暂丢线、弯道/直角/十字下不易误判)。
- 左右轮可闭环、抗电压/负载/摩擦差异。
- 任务通过动作数组/配置自由编排,不改底层控制代码。
- 参数物理意义清晰、分组、限幅、默认合理、低耦合。
- 可调试/可诊断/可回归/可快速定位(传感器 vs 估计 vs 控制 vs 执行器 vs 任务状态机)。
- 保留现有稳定的任务/避障/云台/视觉/OLED 能力,**渐进式重构**。

**非目标(现在不做,避免过度工程化)**

- 不引入 RTOS、不引入复杂脚本解释器。
- 不追求高精度机器人级建图/SLAM。
- 不一次性叠加"编码器闭环 + 十字识别 + 扫描选向 + 参数持久化"。
- 不为"架构漂亮"重写云台/视觉链(其质量已较高)。
- 不移植某一获奖工程的私有实现。

## 4. 目标总体架构

基于当前源码调整后的分层(非机械照搬):

```text
BSP 层        GPIO / (新)Timer PWM / (新)Input Capture / UART / SoftI2C / (新)Flash
Driver 层     Motor / Encoder / TrackSensor / MPU6050 / Ultrasonic / Servo / GimbalStepper
Estimation 层 WheelSpeedEstimator / Odometry / TrackStateEstimator / ImuFilter(新拆分)
Control 层    MotorControl(新) / HeadingControl / LineControl(从 car_controller 抽出) / TurnControl / DistanceControl(新)
Motion 层     CarController(唯一电机出口) / MotionAction / Motion primitives
Mission 层    MissionManager / MissionLibrary / ObstacleAvoidance / GimbalTracker(保留)
Safety 层     SafetySupervisor(新) / FaultManager(新) / Watchdog(新) / EmergencyStop
Debug/UI 层   OLED / Menu / DebugTelemetry(扩展现有 console) / ParamStore(新) / RunSnapshot
```

**层间禁止依赖**

- Control/Estimation 层禁止 include Mission/Task。
- 任务/避障/UI **禁止**直接调用 `Motor_*`(维持现状)。
- Safety 层可越过普通控制链直达"禁止输出",但不得篡改任务进度(只冻结)。

**唯一电机输出**:维持 `CarController` 内 `set_output_speed → Motor_SetSpeed` 为唯一"行驶输出";`Motor_Stop` 为独立零输出旁路。改造后 `set_output_speed` 内部先经 `MotorControl`(闭环)再落到 `Motor_SetSpeed`,出口位置不变。

## 5. 调度与实时性设计

- **保留** 四级节拍(100 µs / 5 ms / 10 ms / 20 ms)与主循环 pending-catchup 结构。
- **P6 关键改造**:设置中断优先级——SysTick(承载步进/PWM 位翻转)应高于 UART RX 与编码器 GPIO 中断,或将电机/舵机 PWM、云台 STEP **迁移到硬件 Timer PWM / 输出比较**,从根本上消除位翻转抖动(见第 6 节)。
- 主环时间预算隔离:OLED 全屏刷新、IMU I2C 读取等长耗时任务应错拍执行,避免挤占同一主循环内 5 ms 云台 / 10 ms 追踪的及时性(pending 上限仅 2,超限丢拍)。
- 新增 **DebugTelemetry 10 Hz** 状态帧(非阻塞,复用视觉 UART TX 或独立 UART),不进 ISR printf(维持现状)。
- 新增 **Watchdog**:20 ms 主环喂狗;主环卡死时硬件复位或 ISR 内超时强制 `Motor_Stop`。

## 6. 电机与硬件 PWM 设计

**原则:先保留 `Motor_SetSpeed` 逻辑量纲(−1000..+1000),只改底层实现。**

- 迁移软件 PWM → 硬件 Timer PWM:
  - 明确 PWM 频率(建议 ≥10 kHz,消除 100 Hz 啸叫/低速死区)、计数周期、有效占空比范围、死区。
  - 底层做"逻辑命令(−1000..1000)→ 硬件 compare value"映射,上层命令语义不变。
  - **方向切换顺序**:反转前先输出 0,等待保护时间(死区),再切方向位,再上占空比。
  - 左右轮方向反转沿用 `MOTOR_A/B_INVERT_DIRECTION` 宏配置。
- **资源冲突审查(待确认)**:硬件 Timer 通道不得与云台步进、转向舵机、超声波冲突;若通道不足,云台步进可保留软件位翻转但提升 SysTick 优先级。
- 移除软件 PWM 时**只删 `Motor_PwmTick100us` 对应 tick**,不得误删共用 SysTick 的 Gimbal/Servo/Ultrasonic tick。
- 停车语义:评估"滑行 vs 主动刹车"(H 桥双高刹车)对急停距离的影响(待确认赛道要求)。
- **功能开关**:`FEATURE_HW_PWM` 编译期开关,台架验证通过前保留软件 PWM 快速回退。

## 7. 编码器、速度估计与里程设计

**前置待确认**:PPR、轮径、齿比必须实测。以下为设计框架。

## 8. MotorControl 速度闭环设计

**独立左右轮速度控制层(新增),第一版用可解释方案:前馈 + PI + 输出限幅 + 抗积分饱和。**

```text
CarController 输出 left/right 目标轮速(cm/s)
MotorControl 读取 WheelSpeedEstimator 实际轮速
MotorControl 输出最终 PWM 命令(−1000..1000)→ set_output_speed
```

运行状态机与更新顺序(每 20 ms):

1. 读实际轮速(valid?否→降级开环,用目标速度直接映射占空比)。
2. 目标速度斜坡(限加速度,防冲击)。
3. 误差 = 目标 − 实际;`u = Kff*目标 + Kp*误差 + Ki*∫误差`。
4. 输出限幅 `±1000`;**抗积分饱和**:输出饱和时冻结/回退积分。
5. 静摩擦/启动死区补偿(低速加最小占空);正反转切换保护(先 0 再换向)。
6. 零目标:积分清零并输出 0(不靠积分维持)。
7. `SafetyHold` → 立即 PWM=0(维持 `set_output_speed` 顶部旁路)。
8. 暂停/继续/避障接管/动作切换:**重置积分与斜坡状态**(复用现有解除 Hold 复位路径)。

- 复用现有 `pid.c`(已含抗饱和/限幅/Reset)🔵→改为真正使用。
- 降级策略:编码器 `error/stale` → 关闭该轮闭环,回退开环,并置 Fault 标志上报。
- **堵转检测**:目标非零且实际轮速≈0 持续 N ms → Fault;**超速检测**:实际远超目标 → Fault。
- **功能开关** `FEATURE_SPEED_LOOP`:先单轮台架、再双轮直行、最后进巡线;可一键回退开环。

## 9. 稳定巡线与赛道状态机设计

### 9.1 TrackSensor Runtime(新增历史状态)

```text
raw / filtered_raw / last_raw / active_count
error / filtered_error / last_valid_error / confidence
lost_ms / all_black_ms / abnormal_ms
turn_candidate / turn_confirmed / track_state
```

- 修复误差计算:除以权重和(或保持除以个数但补死区),丢线时**不立即归零**,进入短时保持。
- 微分项加低通;误差量化台阶(权重步长 100)导致的单帧跳变需滤波后再进 KD。

### 9.2 赛道状态机

```text
NORMAL / LEFT_EDGE / RIGHT_EDGE / SHARP_LEFT / SHARP_RIGHT
CROSS(新增) / ALL_BLACK / TEMP_LOST / LOST / REACQUIRE / INVALID
```

- **不凭单帧进入关键状态**:时间防抖、进入/退出双阈值、候选→确认、传感器置信度。
- 区分**短暂丢线(TEMP_LOST,用 last_valid_error,限时窗口)** 与 **真实丢线(LOST,超时转 LOST_RECOVER)**——落地当前只写不读的 `last_valid_error` 与未用的 `lost_line_threshold`。
- 新增 **CROSS/全黑识别**(当前缺失 ⚪)。

### 9.3 LineControl(从 car_controller 抽出)

```text
base_speed
+ 曲率/误差自适应降速(大误差降 base,直线提速)
+ 横向误差控制(解耦转向修正)
+ 输出限幅(饱和时保持左右差速对称)
→ left_target_cmps / right_target_cmps
```

- 解耦 `base_speed` 与转向修正,消除 base/max_correction/KP/KD 强耦合。
- 输出改为**目标轮速**交给 MotorControl,而非直接差速占空比。
- **参数标定顺序**:KP → KD(带滤波)→ max_correction → base_speed → 自适应降速,一次只调一组。

### 9.4 丢线恢复与特殊路口

- 短时丢线保持 → 按 last_valid 方向搜索 → 超时扩大搜索 → 直角识别 → 十字/大面积黑线识别 → 回线稳定窗(沿用现有 REACQUIRE 两阶段 SEARCH/SETTLE 🟢)。
- 任务层策略:"只报告路口(REPORT_ONLY)" vs "底层自动处理(AUTO)"并存,维持现有 `turn_policy`。

## 10. IMU、航向与转角控制设计

- **拆分职责**:SoftI2C/BSP → MPU6050 寄存器驱动 → 原始校验 → 零偏标定 → ImuFilter(滤波+尖峰拒绝)→ Yaw 积分 → HeadingControl → TurnControl。
- 用**真实 dt**(由 SysTick 计数得到控制环实际间隔),替代硬编码 0.02;读失败帧要么跳过积分并标 stale,要么用上次 dt 外推(明确策略)。
- 上电静止标定保留 🟢;明确零偏/温漂/长期漂移边界;低通只抑高频,不宣称根治长期漂移。
- **统一角度归一化**函数处理 ±180°(现有 `normalize_*` 合并为单一实现)。
- IMU 未就绪:依赖 IMU 的动作明确失败/降级(维持现状 🟢),不静默继续。
- 航向输出最终转为**左右目标轮速**(经 MotorControl),不直接 PWM。
- 相对转角 / 绝对参考角 / 任务起始角明确参考系(现有 mission_start_yaw 保留)。
- **TURN_TO_YAW 状态机(修 P-1)**:

```text
阶段:FAST(|err|>快阈值,turn_speed) → SLOW(比例减速,替代两级 bang-bang) → SETTLE(|err|≤容差持续 T)
新增:超调抑制(可加反向制动)、停稳判定、【超时保护 + 失败码】(当前缺失,最高优先补齐)
```

## 11. 运动原语与任务编排设计

## 12. 避障、安全与故障管理设计

**新增轻量 SafetySupervisor / FaultManager**,统一处理:用户暂停、任务取消、前方紧急障碍、普通障碍保持、避障临时接管、编码器失效、IMU 失效、传感器异常、电机堵转、动作超时、看门狗/主环卡死。

**控制优先级(显式实现,替代现有隐式顺序)**:

```text
EmergencyStop > FaultStop > ObstacleSafetyHold > Avoidance Override > Manual Pause > Mission Command
```

- `SafetyHold` = **禁止最终输出**(维持现状 🟢),非目标置零。
- `ExternalHold` = 只冻结任务时间与动作推进(维持现状 🟢)。
- 避障完成后经 `MotionAction_ReapplyControllerTarget` 重下发原动作(维持现状 🟢)。
- 解除 Hold 防旧积分/旧目标/旧 PWM 冲出(维持现有复位路径 🟢)。
- **修 P-2**:避障失败升级为 `CAR_STATE_ERROR`(可解释、可观察、默认安全),而非永久静默 Hold。
- **修 P-8**:新增独立**软件急停通道**(按键/UART 命令)接入优先级链最高位。
- **修 P-4/P-3**:FaultManager 接入看门狗、编码器 stale、堵转检测。

## 附:确认为工程亮点、必须保留的设计

- 唯一电机输出出口(`Motor_SetSpeed@car_controller.c:106`)。

- 视觉 40 字节 CRC 定长帧 + 非阻塞环形缓冲 RX + 字节级重同步 + 双轴追踪状态机 + 文本调参 REPL。
