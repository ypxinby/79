# 云台运控文档

文档定位：当前工程二维云台运控基线说明  
适用对象：后续继续优化云台控制、视觉追踪、OLED调试和参数标定的开发人员  
代码基线：当前实际代码  
注意：本文档只描述当前工程中云台相关的真实实现，不恢复旧设计，不采用已废止的旧运控逻辑。

## 1. 当前云台总体结构

当前云台为二维步进电机云台：

```text
VisionReceiver / 菜单 / 任务层
        ↓
GimbalVisionAdapter
        ↓
YAW Vision Tracker / PITCH Vision Tracker
        ↓
gimbal.c 公开接口
        ↓
YAW低层 GimbalStepper / PITCH低层STEP调度
        ↓
STEP / DIR / EN
        ↓
二维云台
```

核心边界：

- 任务层、协议层、视觉层不得直接操作 `STEP/DIR/EN`；
- YAW视觉追踪使用 wrapped目标角积分，再走现有YAW位置控制；
- PITCH视觉追踪只使用连续速度接口 `Gimbal_PitchSetTrackingSpeedDegS()`；
- PITCH视觉追踪禁止用位置目标模拟速度；
- YAW/PITCH底层均使用100us tick生成STEP脉冲；
- 需要人工确认零点后，自动位置/视觉控制才允许运行。

### 2.1 YAW步进电机引脚

当前代码中，YAW低层由 `GimbalStepper` 驱动。注意：SysConfig里这组名字仍带 `PITCH_*` 历史命名，但当前代码实际把它作为YAW使用。

| 当前用途 | 代码宏 | MCU GPIO | IOMUX | package pin | 初始状态 | 说明 |
| --- | --- | --- | --- | --- | --- | --- |
| YAW STEP | `GPIO_GIMBAL_YAW_STEP_PIN` | `GPIOB.4` | `IOMUX_PINCM17` | 52 | low | 由100us tick翻转产生脉冲 |
| YAW DIR | `GPIO_GIMBAL_YAW_DIR_PIN` | `GPIOA.1` | `IOMUX_PINCM2` | 34 | low | `GIMBAL_STEPPER_POSITIVE_DIR_HIGH=0` |
| YAW EN | `GPIO_GIMBAL_YAW_EN_PIN` | `GPIOA.2` | `IOMUX_PINCM7` | 42 | set/release | `GIMBAL_EN_ACTIVE_LOW=0`，高电平使能 |

历史命名对应：

| SysConfig名称 | 当前实际用途 |
| --- | --- |
| `GPIO_GIMBAL_A_PITCH_DIR` | YAW DIR |
| `GPIO_GIMBAL_A_PITCH_EN` | YAW EN |
| `GPIO_GIMBAL_B_PITCH_STEP` | YAW STEP |

后续如整理SysConfig，建议把这组名称改成 `YAW_*`，但改名前要同步检查所有生成宏和代码兼容宏。

## 3. 时基和调度

主时基来自SysTick：

```text
APP_TICK_HZ = 10000 Hz
基础tick = 100us
```

调度关系：

| 周期 | 入口 | 作用 |
| --- | --- | --- |
| 100us | `SysTick_Handler()` | 电机PWM、YAW/PITCH STEP tick、舵机、超声波 |
| 5ms | `Gimbal_Update5ms()` | 两轴速度斜坡、RPM到STEP半周期换算 |
| 10ms | `GimbalVisionYawTracker_Update10ms()` | YAW视觉追踪计算 |
| 10ms | `GimbalVisionPitchTracker_Update10ms()` | PITCH视觉追踪计算 |
| 10ms | `GimbalTracker_Update(0.010f)` | legacy P23X/P23V调试Tracker |
| 20ms | `App_Update_20ms()` | 菜单、OLED、小车任务等应用层更新 |

主循环中，视觉UART处理在低优先级执行：

```text
VisionUart_Process()
VisionReceiver_Process()
GimbalVisionAdapter_Update()
10ms Tracker更新
5ms Gimbal更新
20ms App更新
```

实时性原则：

- 100us中断只做确定性短任务；
- UART ISR只搬运字节到接收结构；
- CRC、协议解析、OLED刷新、Tracker调用不在UART ISR中执行；
- 后续优化时不要在100us tick里加入浮点、大量循环或OLED输出。

## 4. 坐标、单位和换算

步进参数：

```text
YAW steps/rev   = 3200
PITCH steps/rev = 3200
360 deg/rev
内部角度常用 deg_x10，即 0.1°单位
```

换算：

```text
deg_x10 ↔ steps
steps = deg_x10 * 3200 / 3600
1 rpm = 6 deg/s
```

速度相关：

```text
GIMBAL_TICK_HZ = 10000
STEP半周期tick = 100us个数
低于 GIMBAL_MIN_EFFECTIVE_RPM = 0.5 rpm 时认为无有效输出
```

反馈字段中常见单位：

| 字段 | 单位 |
| --- | --- |
| `continuous_deg_x10` | 连续角度 ×10 |
| `wrapped_deg_x10` | 0~360 wrapped角度 ×10 |
| `target_deg_x10` | 目标角度 ×10 |
| `commanded_rpm_x10` | 实际斜坡后RPM ×10 |
| `step_half_period_ticks` | STEP半周期tick，1 tick=100us |

## 5. YAW控制架构

YAW底层模块：

```text
gimbal.c
  ↓
GimbalStepper
  ↓
GPIOA.1 DIR / GPIOA.2 EN / GPIOB.4 STEP
```

YAW公开接口：

| 接口 | 作用 |
| --- | --- |
| `Gimbal_YawConfirmZero()` | 人工确认当前位置为线缆中立零点 |
| `Gimbal_YawRelease()` | 释放电机并清除 `position_valid` |
| `Gimbal_YawStopHold()` | 停止并保持当前位置 |
| `Gimbal_YawMoveRelativeDeg()` | 相对移动 |
| `Gimbal_YawMoveToDeg()` | 连续角绝对目标 |
| `Gimbal_YawMoveWrappedDeg()` | wrapped角目标，选择可行最短路径 |
| `Gimbal_YawSetWrappedTargetDeg()` | 实时刷新wrapped目标，不主动重置速度斜坡 |
| `Gimbal_YawEnableWorldLock()` | 世界锁定 |
| `Gimbal_YawDisableWorldLock()` | 关闭世界锁定 |

当前YAW策略：

```text
GIMBAL_YAW_HAS_SLIP_RING = 0
无滑环
上电后必须人工确认YAW零点
连续角软限位启用
限位范围：-170° ~ +170°
```

YAW位置控制流程：

```text
目标角度
↓
连续角/ wrapped角选择
↓
软限位检查
↓
换算为目标steps
↓
GimbalStepper_MoveToEstimatedSteps()
↓
5ms速度斜坡
↓
100us STEP输出
```

YAW速度参数：

```text
普通位置最大速度：GIMBAL_MAX_POSITION_RPM = 4 rpm = 24 deg/s
世界锁定最大速度：GIMBAL_MAX_LOCK_RPM = 12 rpm = 72 deg/s
普通加速度限制：10 rpm/s
世界锁定加速度限制：30 rpm/s
```

世界锁定逻辑：

- 启用时记录 `IMU yaw + 当前YAW连续角` 作为锁定世界角；
- 5ms更新中根据IMU yaw刷新YAW目标；
- YAW视觉追踪V1与世界锁定互斥；
- world lock开启时，YVT拒绝启用并显示 `WLK`。

## 6. PITCH控制架构

PITCH底层在 `gimbal.c` 内部实现：

```text
gimbal.c PITCH模块
  ↓
GPIOB.6 DIR / GPIOB.7 EN / GPIOB.5 STEP
```

PITCH公开接口：

| 接口 | 作用 |
| --- | --- |
| `Gimbal_PitchConfirmZero()` | 人工确认当前位置为机械中心0° |
| `Gimbal_PitchRelease()` | 释放电机并清除 `position_valid` |
| `Gimbal_PitchStopHold()` | 停止并保持当前位置 |
| `Gimbal_PitchMoveRelativeDeg()` | 相对角度移动 |
| `Gimbal_PitchMoveToDeg()` | 位置模式移动 |
| `Gimbal_PitchSetTargetDeg()` | 位置目标更新 |
| `Gimbal_PitchSetTrackingSpeedDegS()` | 视觉追踪连续速度接口 |
| `Gimbal_PitchStopTrackingHold()` | 追踪停止保持 |

当前PITCH策略：

```text
上电后 position_valid = 0
需要人工摆到机械中心后确认零点
软件限位：-50° ~ +50°
视觉追踪只允许使用连续速度接口
```

PITCH控制模式：

```c
GIMBAL_PITCH_CONTROL_POSITION
GIMBAL_PITCH_CONTROL_TRACK_SPEED
```

PITCH速度参数：

```text
最大位置/追踪底层速度：4 rpm = 24 deg/s
加速度限制：10 rpm/s
视觉PITCH默认MAXSPD：6 deg/s
视觉PITCH最小非零追踪速度：3 deg/s
```

PITCH限位行为：

- 位置模式目标会被夹紧到 `-50°~+50°`；
- 速度追踪模式接近限位时停止输出并设置 `limit_direction`；
- 到达限位后，反方向命令允许退出限位；
- `position_valid=0` 时，PITCH自动位置/视觉控制会被拒绝或停止。

## 7. 零点和安全状态

### 7.1 YAW零点

YAW无滑环，当前以线缆安全为优先：

```text
上电后 position_valid=0
手动调整到线缆中立位置
在YAW页面长按K2确认零点
确认成功后 YAW=0°
```

YAW Release：

- 释放电机；
- 清除 `position_valid`；
- 禁用world lock；
- 后续自动控制必须重新确认零点。

### 7.2 PITCH零点

PITCH以机械中心为0°：

```text
上电后 position_valid=0
手动调整到机械中心
在PITCH页面长按K2确认零点
确认成功后 PITCH=0°
```

PITCH Release：

- 释放电机；
- 清除 `position_valid`；
- 后续PITCH位置/视觉控制必须重新确认零点。

### 7.3 StopHold和Release区别

| 操作 | 电机使能 | 位置有效性 | 用途 |
| --- | --- | --- | --- |
| StopHold | 保持使能 | 保持有效 | 停止运动但维持当前位置 |
| Release | 关闭使能 | 清除有效 | 允许手动挪动，位置估算不再可信 |

### 9.3 参数含义

| 参数 | 含义 | 过小现象 | 过大现象 |
| --- | --- | --- | --- |
| `DB` | 中心死区，单位px | 容易抖动 | 居中误差变大 |
| `KP` | 误差到速度比例 | 跟随慢 | 抖动、过冲 |
| `MAXSPD` | 最大追踪速度 | 跟不上 | 运动冲、可能失步 |
| `TIMEOUT` | 视觉数据超时保护 | 偶发误停 | 断流后停得慢 |

当前不支持Flash保存，复位后恢复默认参数。

```text
YVT:<state> E<enabled> Z<position_valid>
EX:<error_x> S:<yaw speed>
T:<target_wrapped> C:<current_wrapped>
L:<limit_direction> P<positive_limit> N<negative_limit>
```

VPT页面：

```text
VPT DB<deadband> TO<timeout>
KP<kp> MAX<max_speed>
ALPHA<filter_alpha>
L:<last change>
```

## 11. 关键保护和互斥关系

| 场景 | 当前处理 |
| --- | --- |
| YAW未校零 | YAW自动位置/视觉控制拒绝运行 |
| PITCH未校零 | PITCH自动位置/视觉控制拒绝运行 |
| YAW world lock开启 | YVT拒绝运行 |
| legacy Tracker启用 | YVT/PVT会停止并进入PRE |
| 视觉valid=0 | YVT/PVT停止保持 |
| 视觉超时 | YVT/PVT停止保持 |
| YAW到线缆软限位 | 禁止继续向限位外积分，允许反向退出 |
| PITCH到±50°限位 | 禁止继续向限位外运动，允许反向退出 |
| Release | 清除对应轴 `position_valid` |

## 13. 后续优化开发注意事项

优先遵守：

1. 当前实际代码；
2. 当前工程文档；
3. 实机验收结果；
4. 历史旧文档只作原理参考。

禁止回退到已废止设计：

- 不要把PITCH视觉追踪改成位置目标积分；
- 不要把PITCH当连续360°轴；
- 不要为了统一架构重写YAW和PITCH；
- 不要让任务层、协议层直接操作 `STEP/DIR/EN`；
- 不要直接采用旧文档中的高速度参数。

建议后续优先项：

- 将YAW参数也加入OLED显示页，或扩展VPT为轴选择显示；
- 统一YVT `TMO` 与PVT `OLD` 的命名，不改变行为；
- 对双轴长时间运行做100us ISR最坏时间测量；
- 若未来确认安装滑环，再重新评估YAW连续角限位策略；
- 若整理SysConfig命名，先做引脚宏兼容迁移，避免YAW/PITCH接线混淆；
- 参数保存到Flash前，先定义版本号、CRC和默认恢复策略。
