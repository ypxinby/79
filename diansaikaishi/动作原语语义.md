# 动作原语语义设计

> 目的：用尽量少的代码，把几个简单动作封装成积木，后续拼出第一版可工作的主动避障流程。本文只定义第一版轻量方案，不追求高精度通用机器人控制。

## 1. 第一版目标

主动绕障先理解成三块积木：

```text
TURN_TO_YAW
按照 IMU 相对转一个角度

DRIVE_HEADING_TIME
保持动作启动时的方向直行一段时间

REACQUIRE_LINE
寻找黑线，找到后稳定循迹一小段
```

左绕示例：

```text
向左转
-> 斜着直行
-> 转回原方向
-> 直行越过障碍
-> 向右转找线
-> REACQUIRE_LINE
-> 恢复原 FOLLOW 任务
```

右绕完全镜像。

第一版最重要的是：

```text
转向能重复
直行基本稳定
回线不误判
动作能够组合
失败时能够停车
```

## 2. 保持的分层

继续保持当前工程边界：

```text
MissionManager      负责原地图任务
MotionAction        负责动作启动和完成判断
ObstacleAvoidance   后续负责简单绕障状态机
CarController       统一负责电机输出
ObstacleSafety      负责普通停车和紧急停车
ObstacleScanner     只给 L/R/D 推荐方向
```

任何新动作都不能直接调用 `Motor_SetSpeed()`，电机输出仍然统一经过 `CarController`。

## 3. TURN_TO_YAW

### 3.1 语义

第一版只支持相对转角：

```c
ACTION_TURN_RELATIVE_YAW(+40.0f, 2000U)
ACTION_TURN_RELATIVE_YAW(-40.0f, 2000U)
```

含义：

```text
target_yaw = action_start_yaw + angle_deg
```

正角度代表左转还是右转，由实车 IMU yaw 正方向统一确定。任务数组不要为了适配方向到处反写角度，如果方向相反，只在 `CarController` 内统一修正。

### 3.2 yaw 误差

误差必须归一化：

```text
error = normalize(target_yaw - current_yaw)
范围：-180 deg ~ +180 deg
```

不能直接用 `target_yaw - current_yaw` 做完成判断，否则跨越 +180/-180 时可能转错方向。

### 3.3 最简速度控制

第一版只做两段速度：

```text
|error| > 12 deg    使用 turn_speed
|error| <= 12 deg   使用 slow_turn_speed
```

如果暂时不增加新参数，`slow_turn_speed` 可以用 `turn_speed` 的比例计算，但要保证电机能动。

第一版暂缓：

```text
复杂 PID
三段以上速度控制
角速度稳定判断
电机堵转判断
绝对 yaw 动作宏
```

### 3.4 完成条件

最简完成条件：

```text
abs(error) <= 5 deg
连续 5 个 20ms 周期满足
-> 完成
```

也就是稳定约 100ms。

完成时：

```text
停车
MotionAction 返回 SUCCESS
```

### 3.5 失败条件

```text
IMU 未就绪
超过 timeout_ms 仍未完成
```

失败时：

```text
停车
MotionAction 返回 FAILED 或 TIMEOUT
```

## 4. DRIVE_HEADING_TIME

### 4.1 语义

动作启动时记录当前 yaw：

```text
target_yaw = current_yaw
```

然后保持这个方向直行固定时间：

```c
ACTION_DRIVE_HEADING_TIME(700U, 1200U)
```

含义：

```text
保持启动时方向直行 700ms
最长允许运行 1200ms
```

### 4.2 输出

复用现有 `HeadingControl` 修正左右轮：

```text
left  = speed + correction
right = speed - correction
```

动作期间不做灰度循迹修正。

### 4.3 完成条件

```text
elapsed_ms >= duration_ms
-> 完成
```

第一版不做：

```text
编码器距离闭环
按厘米前进
提前检测回线
自动切换目标 yaw
复杂动作反馈字段
```

后续需要提高稳定性时，再加：

```text
DRIVE_HEADING_DISTANCE
```

## 5. REACQUIRE_LINE

### 5.1 语义

`REACQUIRE_LINE` 不是新的底层运动模式，它只是：

```text
SEEK + FOLLOW 300ms
```

因此第一版不新增：

```c
TRACK_MODE_REACQUIRE_LINE
```

职责划分：

```text
CarController 负责已有 SEEK/FOLLOW
MotionAction 负责 REACQUIRE_LINE 的 SEARCH/SETTLE 流程
```

### 5.2 SEARCH

直接复用：

```c
CarController_StartSeekLine(target_yaw)
```

第一版 target_yaw 使用动作启动时当前方向：

```text
target_yaw = current_yaw
```

进入 SETTLE 的条件：

```text
中间传感器连续 3 个 20ms 周期看到线
```

不要任意一个边缘传感器看到线就结束，否则绕障时容易误判。

### 5.3 SETTLE

进入 SETTLE 后调用：

```c
CarController_StartFollowLine(TURN_POLICY_IGNORE)
```

稳定循迹：

```text
300ms 内没有确认丢线
-> 回线成功
```

完成后，后续由任务恢复接口重新下发原 FOLLOW 策略。

### 5.4 失败条件

```text
超过 timeout_ms 仍未成功回线
SETTLE 阶段再次确认丢线
CarController 进入 ERROR
```

失败时：

```text
停车
MotionAction 返回 FAILED 或 TIMEOUT
```

## 6. 第一版代码增量

### 6.1 MotionAction

只新增三个动作类型：

```c
MOTION_ACTION_TURN_TO_YAW,
MOTION_ACTION_DRIVE_HEADING_TIME,
MOTION_ACTION_REACQUIRE_LINE
```

对应宏：

```c
ACTION_TURN_RELATIVE_YAW(angle_deg, timeout_ms)
ACTION_DRIVE_HEADING_TIME(duration_ms, timeout_ms)
ACTION_REACQUIRE_LINE(timeout_ms)
```

第一版不要增加大量 yaw 参考类型。简单语义固定为：

```text
TURN_TO_YAW         相对于动作开始 yaw 转动
DRIVE_HEADING_TIME  保持动作开始 yaw
REACQUIRE_LINE      保持当前方向找线
```

### 6.2 CarController

只新增两个运行模式：

```c
TRACK_MODE_TURN_TO_YAW
TRACK_MODE_DRIVE_HEADING
```

不新增：

```c
TRACK_MODE_REACQUIRE_LINE
```

第一版尽量复用现有 feedback：

```text
turn_completed    可用于 TURN_TO_YAW 完成
operation_failed  用于动作失败
line_found/line_lost/center_detected 用于 REACQUIRE_LINE 判断
```

暂缓大量独立字段：

```text
drive_heading_completed
reacquire_line_completed
reacquire_line_failed
```

## 7. 简单主动绕障控制权

第一版不做复杂 Owner 枚举，不做双 MotionAction 运行时。

继续使用现有：

```text
ExternalHold
SafetyHold
```

额外只需要一个简单状态：

```c
bool avoidance_active;
```

状态含义：

```text
正常任务：
ExternalHold = false
SafetyHold = false
avoidance_active = false

障碍停车扫描：
ExternalHold = true
SafetyHold = true
avoidance_active = false

开始主动绕障：
ExternalHold = true
SafetyHold = false
avoidance_active = true

避障失败：
ExternalHold = true
SafetyHold = true
avoidance_active = false
```

含义：

```text
原任务冻结
绕障期间允许 ObstacleAvoidance 驱动车辆
电机输出仍然经过 CarController
失败或危险时重新停车
```

## 8. 普通 O:1 和紧急停车

主动绕障期间，原障碍可能仍然让 `O:1` 保持。如果普通 `O:1` 继续触发 `SafetyHold`，车永远动不了。

第一版简化规则：

```c
if (!avoidance_active) {
    safety_hold = obstacle_blocked;      /* 例如 20cm */
} else {
    safety_hold = emergency_blocked;     /* 例如 10cm */
}
```

也就是：

```text
正常任务：20cm 内停车
主动绕障：允许绕原障碍运动，但 10cm 内紧急停车
```

不需要复杂安全模式枚举。

## 9. 第一版只允许 FOLLOW 中主动绕障

这是重要简化条件：

```text
FOLLOW_LINE 遇到障碍：允许扫描并主动绕障
SEEK_LINE 遇到障碍：只停车等待
TURN_90 遇到障碍：不启动主动绕障
LOST_RECOVER 遇到障碍：只停车等待
```

这样绕障完成后只需要恢复：

```text
FOLLOW_LINE
```

暂时不用处理 SEEK、TURN、LOST 等复杂恢复场景。

## 10. 恢复原 FOLLOW 动作

绕障完成后不能只解除：

```c
MissionManager_SetExternalHold(false)
```

因为 `CarController` 可能仍处在绕障动作留下的模式。

但第一版只允许 FOLLOW 中绕障，所以恢复可以简化为：

```c
void MotionAction_ReapplyControllerTarget(void);
```

职责：

```text
不重新开始动作
不清零 action elapsed time
只根据当前 MotionAction 重新下发 CarController 目标
```

例如当前任务是：

```text
FOLLOW_UNTIL_RIGHT_90
```

恢复时只需要重新调用等价逻辑：

```c
CarController_StartFollowLine(CAR_TURN_POLICY_REPORT_ONLY)
```

然后：

```text
解除 ExternalHold
MissionManager 继续原 action
```

## 11. ObstacleAvoidance 第一版状态机

后续新增：

```text
obstacle_avoidance.c
obstacle_avoidance.h
```

状态控制在：

```c
typedef enum {
    AVOID_IDLE = 0,
    AVOID_WAIT_SCAN,
    AVOID_TURN_OUT,
    AVOID_DRIVE_OUT,
    AVOID_TURN_FORWARD,
    AVOID_DRIVE_PASS,
    AVOID_TURN_TO_LINE,
    AVOID_REACQUIRE,
    AVOID_COMPLETE,
    AVOID_FAILED
} ObstacleAvoidState;
```

每个状态只做：

```text
进入时启动动作
每 20ms 判断是否完成
完成后进入下一状态
失败则停车
```

不做通用双动作运行时。

## 12. 暂缓内容

第一版全部暂缓：

```text
绝对 yaw 动作宏
大量 yaw reference 枚举
TRACK_MODE_REACQUIRE_LINE
drive_heading_completed / reacquire_line_completed 等独立 feedback
MotionActionRuntime 双实例
MotionControlOwner
复杂错误枚举
gyro 稳定判定
电机堵转检测
编码器距离闭环
按厘米行驶
SEEK/TURN/LOST 场景主动绕障
```

这些设计不是不好，只是现在没有必要。

## 13. 测试任务

必须先单独测试三个动作。

```text
TEST-YAW:
  TURN_RELATIVE_YAW(+40)
  WAIT
  TURN_RELATIVE_YAW(-40)
  STOP

TEST-HEAD:
  TURN_RELATIVE_YAW(+40)
  DRIVE_HEADING_TIME(500~1000ms)
  STOP

TEST-REQ:
  REACQUIRE_LINE
  FOLLOW_FOREVER
```

通过标准：

```text
TURN_TO_YAW 不跨 180 度转错方向
TURN_TO_YAW 到位后不过度震荡
DRIVE_HEADING_TIME 脱线直行方向基本稳定
REACQUIRE_LINE 不因边缘偶然扫线提前完成
REACQUIRE_LINE SETTLE 后能稳定进入 FOLLOW
失败和超时都能停车
```

## 14. 推荐实施顺序

```text
1. 只做 TURN_TO_YAW
2. 建 TEST-YAW 验证 ±40 deg
3. 做 DRIVE_HEADING_TIME
4. 建 TEST-HEAD 验证直行 500~1000ms
5. 做 REACQUIRE_LINE
6. 建 TEST-REQ 验证找线和稳定循迹
7. 新建简单 ObstacleAvoidance 状态机
8. 只支持 FOLLOW 中主动绕障
9. 最后接入 L/R/D 自动选择方向
10. 后续再做 DRIVE_HEADING_DISTANCE
```
