# 工程开发文档 2026-07-10 巡线稳定版

> 当前版本已经完成七路灰度循迹、OLED 调试、按键调参、90 度转弯识别、单轮转弯和稳定的转弯后找线恢复。
> 本文档按当前代码实际状态整理，作为后续继续开发的基准说明。

---

# 一、当前系统状态

## 1. 已实现功能

- 20ms 主循环调度，控制算法不放在中断里执行。
- TB6612 双电机软件 PWM 输出。
- 七路循迹传感器读取，统一转换为 `raw bit = 1` 表示检测到黑线。
- 基于七路权重的误差计算。
- 正常巡线 PD 控制。
- 启动找线 `SEEK_LINE`。
- 丢线恢复 `LOST_RECOVER`。
- 90 度左弯、右弯识别。
- 90 度单轮转弯动作。
- 转弯后按最后一次有效偏差方向单方向找线。
- OLED 三页显示：状态页、参数页、传感器页。
- 三按键控制启动、暂停、急停、参数切换和参数调整。
- CCS Expressions 可观察核心调试变量。

## 2. 本版重要变更

- 已取消 `motor_balance` / `TRIM` 左右轮静态补偿。
- 找线阶段也不再加入左右轮 balance 补偿。
- 后续直线稳定性不再靠固定 balance 修正，计划改用陀螺仪航向闭环。
- `LOST_RECOVER` 不再左右来回扫线，而是根据 `recover_direction` 固定方向找线；无可靠方向时低速直走找线。
- 当前 90 度转弯已实测可以稳定转弯。

## 3. 当前未完成或预留功能

- `current_lap`、`target_laps`、`start_line_threshold`、`lap_cooldown_ms` 已有字段或菜单基础，但计圈逻辑尚未接入 `CarController`。
- `lost_line_threshold` 当前可在菜单显示和修改，但当前丢线判断仍是 `raw == 0`，没有使用该阈值。
- 陀螺仪模块尚未接入。
- 旧 `straight_control.c/h` 编码器直行模块已经清理；通用 `pid.c/h` 保留，后续可复用于陀螺仪航向控制。

# 二、系统架构

## 1. 文件职责

| 文件 | 职责 |
|---|---|
| `empty.c` | 程序入口、SysTick、GPIO 中断入口、20ms 调度标志 |
| `app.c/h` | 全系统初始化和 20ms 应用层调度 |
| `app_config.c/h` | 全局参数默认值、上下限保护 |
| `car_state.c/h` | 整车状态：MENU、READY、RUN、PAUSE、DONE、ERR |
| `car_controller.c/h` | 巡线状态机、PD 控制、90 度转弯、丢线恢复、电机目标输出 |
| `track_sensor.c/h` | 七路传感器读取、误差计算、90 度检测 |
| `motor.c/h` | TB6612 电机方向和软件 PWM |
| `key.c/h` | 三按键消抖、短按、长按事件 |
| `menu.c/h` | OLED 页面切换、参数项切换、参数修改 |
| `oled.c/h` | OLED 底层模拟 I2C 和字符显示 |
| `oled_ui.c/h` | OLED 页面内容渲染 |
| `encoder.c/h` | 编码器计数接口，当前未参与巡线速度闭环 |
| `pid.c/h` | 通用 PID 控制器，保留给后续陀螺仪航向控制等模块复用 |

## 2. 调度结构

`SysTick_Handler()` 每 100us 执行一次：

```text
Motor_PwmTick100us()
累计到 20ms 后置位 g_appUpdateDue
```

主循环发现标志后执行：

```text
App_Update_20ms()
```

当前 20ms 周期内执行顺序：

```text
1. Key_Update_20ms()
2. Key_GetEvent()
3. Menu_HandleKeyEvent()
4. CarController_Update_20ms()
5. 更新 CCS 调试变量
6. OledUi_Update_20ms()
```

设计原则：

- 中断只做 PWM tick、编码器计数和调度置位。
- 按键、OLED、巡线控制都在主循环中执行。
- OLED 只显示状态，不参与控制计算。
- `CarController` 是当前唯一实际输出电机速度的运控模块。

# 三、传感器设计

## 1. RAW 位定义

七路传感器从左到右：

```text
S1 S2 S3 S4 S5 S6 S7
```

对应 raw bit：

```text
bit0 = S1
bit1 = S2
bit2 = S3
bit3 = S4
bit4 = S5
bit5 = S6
bit6 = S7
```

软件内部统一约定：

```text
bit = 1 表示该路检测到黑线
bit = 0 表示该路未检测到黑线
```

如果实物模块黑白电平相反，只改 `track_sensor.h` 中的 `TRACK_BLACK_LEVEL`，不要改上层算法。

## 2. 误差权重

`track_sensor.c` 中当前权重：

```c
static const int16_t g_trackWeights[TRACK_SENSOR_COUNT] = {
    -300, -200, -100, 0, 100, 200, 300
};
```

含义：

- 线偏左，误差为负。
- 线在中心，误差接近 0。
- 线偏右，误差为正。

多路同时检测到黑线时，误差取所有检测路权重平均值。

## 3. 90 度检测

当前检测规则：

```text
左 90 度：
S1/S2 至少一路有线
S6/S7 没有线
error <= -200

右 90 度：
S6/S7 至少一路有线
S1/S2 没有线
error >= 200
```

返回值：

```text
TRACK_TURN_NONE
TRACK_TURN_LEFT_90
TRACK_TURN_RIGHT_90
```

```text

  -> FOLLOW_LINE

```

# 五、运控设计

## 1. SEEK_LINE

作用：启动后先慢速向前找线。

当前输出：

```c
set_output_speed(search_speed, search_speed);
```

注意：

- 已取消 balance 补偿。
- 左右轮同速前进。
- 一旦任意传感器检测到黑线，进入 `FOLLOW_LINE`。

## 2. FOLLOW_LINE

作用：正常七路循迹。

控制量：

```text
error = TrackSensor_GetErrorFromRaw(raw)
derivative = error - last_error
correction = (track_kp * error + track_kd * derivative) / track_scale
correction 限幅到 +/- max_correction
```

电机输出：

```text
left_speed  = base_speed + correction
right_speed = base_speed - correction
```

注意：

- 当前没有 `motor_balance` 静态补偿。
- 如果车修正方向反了，改 `TRACK_REVERSE_CORRECTION`。
- 后续直线稳定性建议接入陀螺仪闭环，不要重新加入固定 balance。

## 3. TURN_LEFT_90 / TURN_RIGHT_90

作用：检测到 90 度弯后，执行单轮转弯。

当前动作：

```text
L90：
left_speed  = 0
right_speed = turn_speed

R90：
left_speed  = turn_speed
right_speed = 0
```

默认参数：

```text
turn_speed = 130
turn_min_ms = 160
turn_max_ms = 1400
```

退出条件：

```text
turn_elapsed_ms >= turn_min_ms
并且 S3/S4/S5 中间传感器重新检测到黑线
```

保护条件：

```text
turn_elapsed_ms >= turn_max_ms
切入 LOST_RECOVER
```

当前实测结论：

- 该单轮转弯策略已经可以稳定通过 90 度弯。
- 如果后续换电池、电机、电压或场地摩擦，优先调 `turn_speed`，其次调 `turn_min_ms`。

## 4. LOST_RECOVER

作用：丢线后按最后一次明确偏差方向找回线。

本版最新策略：

```text
recover_direction = LEFT：
  认为最后线明确在左侧，向左找线
  left_speed = 0
  right_speed = recover_speed

recover_direction = RIGHT：
  认为最后线明确在右侧，向右找线
  left_speed = recover_speed
  right_speed = 0

recover_direction = NONE：
  没有可靠方向依据，低速直走找线
  left_speed = recover_speed
  right_speed = recover_speed
```

重要说明：

- 当前不再左右来回摆头。
- 没有 `LOST_SWEEP_INTERVAL_MS`。
- `last_valid_error == 0` 不再默认向右找线。
- 只有 `error < 0` 或 `error > 0` 时才刷新 `recover_direction`。
- 左/右 90 度检测会显式写入 `recover_direction`，因此转弯超时进入 `LOST_RECOVER` 时仍有明确方向。
- 如果 `lost_elapsed_ms >= lost_recover_max_ms` 仍未找回线，则停车并进入 `CAR_STATE_ERROR`。

# 九、调参建议

## 1. 正常巡线

推荐起点：

```text
base_speed = 160
track_kp = 100
track_kd = 50
max_correction = 160
```

现象和调整：

```text
蛇形摆动：降低 KP，或提高 KD
转向不够：提高 KP 或 max_correction
修正太猛：降低 KP 或 max_correction
速度太快：降低 base_speed
速度太慢：提高 base_speed
```

## 2. 90 度转弯

推荐起点：

```text
turn_speed = 130
turn_min_ms = 160
turn_max_ms = 1400
```

现象和调整：

```text
转不过去：提高 turn_speed，必要时提高 turn_max_ms
刚进 L90/R90 又退出：提高 turn_min_ms
转过头：降低 turn_speed 或降低 turn_min_ms
迟迟不退出：检查 S3/S4/S5，或降低 turn_min_ms
经常转弯超时进 LOST：提高 turn_max_ms
```

## 3. 丢线恢复

推荐起点：

```text
recover_speed = 100
lost_recover_max_ms = 1400
```

现象和调整：

```text
找线太慢：提高 recover_speed
找线动作太冲：降低 recover_speed
还没找回就 ERR：提高 lost_recover_max_ms
找线方向反了：检查 recover_direction、last_valid_error 符号和左右轮接线
```

# 十、后续开发指导

## 1. 旧直行模块清理状态

旧 `straight_control.c/h` 已经删除，当前工程不再包含早期编码器直行 PID 控制链。

```text
1. app.c 已删除 straight_control.h include
2. App_Init() 已删除 StraightControl_Init()
3. empty.c 中调度标志已改名为 g_appUpdateDue
4. empty.c 中周期宏已改为 APP_UPDATE_PERIOD_MS
5. straight_control.c/h 已删除
```

保留项：

```text
pid.c/h      通用 PID，可给陀螺仪航向闭环复用
encoder.c/h  编码器接口，可后续做速度闭环或里程估计
```

这样后续加入陀螺仪时，工程里不会再出现旧 `StraightControl` 概念混淆。

## 2. 接入陀螺仪直线控制

建议新增模块：

```text
imu.c / imu.h
heading_control.c / heading_control.h
```

建议职责：

```text
imu：
  初始化陀螺仪
  读取角速度
  积分 yaw
  提供零偏校准接口

heading_control：
  保存目标航向 target_yaw
  计算 yaw_error
  输出 heading_correction
```

建议接入方式：

```text
1. App_Init() 初始化 IMU。
2. App_Update_20ms() 或更高频率任务更新 yaw。
3. CarController_ResetRuntime() 清零目标航向或设置起跑航向。
4. 在 FOLLOW_LINE 中只在接近直线时叠加小幅航向修正。
5. 在 TURN_LEFT_90 / TURN_RIGHT_90 中可以先不使用陀螺仪，保持当前稳定方案。
```

建议控制形式：

```text
line_correction = 七路循迹 PD 输出
heading_correction = 陀螺仪航向 PID 输出
final_correction = line_correction + heading_weight * heading_correction

left_speed = base_speed + final_correction
right_speed = base_speed - final_correction
```

注意事项：

- 不要恢复 `motor_balance` 这类固定补偿。
- 陀螺仪用于纠正长期偏航，七路传感器仍负责贴线。
- 起步、90 度转弯、丢线恢复时要明确是否冻结或重置目标航向。
- 先只在直线段启用陀螺仪，确认稳定后再考虑弯道辅助。

## 3. 接入计圈和自动停车

建议利用已有字段：

```text
current_lap
target_laps
start_line_threshold
lap_cooldown_ms
```

建议逻辑：

```text
1. 在 FOLLOW_LINE 中检测 TrackSensor_IsStartLine(raw, start_line_threshold)。
2. 只有 lap_cooldown_ms 结束后才允许计一次圈。
3. 计圈后刷新 lap_cooldown_ms。
4. current_lap >= target_laps 时停车，进入 CAR_STATE_DONE。
```

注意：

- 起跑线和 90 度弯可能都出现多路黑线，阈值要结合实测。
- 计圈只建议在 `FOLLOW_LINE` 中做，不建议在 `TURN` 或 `LOST` 中做。

## 4. 参数持久化

当前参数上电后恢复默认值。后续如果需要现场快速调车，可以加入 Flash 保存：

```text
1. 参数结构体增加版本号和校验。
2. 上电优先读取 Flash 参数。
3. 校验失败则使用 AppConfig_InitDefault()。
4. 长按组合键保存当前参数。
```

## 5. 测试顺序建议

每次新增功能后按以下顺序验证：

```text
1. CCS 编译无错误。
2. 上电 OLED 正常显示。
3. RAW 黑白方向正确。
4. 架空测试左右轮方向。
5. 低速直线循迹。
6. 低速 90 度弯。
7. 丢线恢复方向。
8. 再逐步提高 base_speed。
```

# 十一、风险点

- 90 度检测依赖边缘传感器和误差阈值，不同赛道宽度可能需要调整。
- 单轮转弯对电池电压、地面摩擦和轮胎状态敏感。
- `recover_direction == NONE` 时当前低速直走找线，如果赛道断线较长，可能需要结合陀螺仪航向保持或短延时搜索策略。
- `lost_line_threshold` 当前未参与丢线判断，文档和菜单已有但控制逻辑未接入。
- 陀螺仪接入后要明确航向控制在哪些模式生效，避免和 90 度单轮转弯动作互相干扰。

---

# 十二、当前结论

当前工程主干已经从“能动”进入“可稳定转弯”的阶段。后续开发建议保持现有巡线状态机不大改，优先做三件事：

```text
1. 接入陀螺仪航向闭环，替代固定左右轮补偿。
2. 接入计圈和目标圈数停车。
3. 根据实测再考虑编码器速度闭环或参数 Flash 保存。
```
