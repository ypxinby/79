# 三、七路循迹传感器

## 1. RAW 位定义

```text
bit0 -> S1，最左侧
bit1 -> S2
bit2 -> S3
bit3 -> S4，中间
bit4 -> S5
bit5 -> S6
bit6 -> S7，最右侧
bit7 不使用
```

有效掩码：

```c
#define TRACK_SENSOR_COUNT      (7U)
#define TRACK_RAW_VALID_MASK    (0x7FU)
```

当前黑线电平：

```c
#define TRACK_BLACK_LEVEL       (1)
```

软件约定：

```text
RAW bit = 1 表示检测到黑线
RAW bit = 0 表示白底 / 未检测到黑线
```

## 2. 明天 RAW 测试预期

在 SENSOR 页观察 `RAW`：

```text
白底无黑线: 0000000
中间 S4:     0001000
左侧 S1/S2:  0000011
右侧 S6/S7:  1100000
```

如果白底显示为 1，黑线显示为 0，优先修改：

```c
#define TRACK_BLACK_LEVEL       (0)
```

## 3. 循迹误差权重

```text
S1    S2    S3    S4   S5   S6   S7
-300  -200  -100  0    100  200  300
```

误差方向：

```text
黑线在左侧: error < 0
黑线居中:   error = 0
黑线在右侧: error > 0
```

# 四、运行模式设计

当前 `CAR_STATE_RUNNING` 内部有运行子模式：

```c
typedef enum {
    TRACK_MODE_SEEK_LINE = 0,
    TRACK_MODE_FOLLOW_LINE,
    TRACK_MODE_TURN_LEFT_90,
    TRACK_MODE_TURN_RIGHT_90,
    TRACK_MODE_LOST_RECOVER
} TrackRunMode;
```

OLED 状态页和 SENSOR 页会显示：

```text
M:SEEK
M:LINE
M:L90
M:R90
M:LOST
```

## 1. SEEK_LINE：启动找线

进入条件：

```text
READY 下短按 K2 启动
CarController_ResetRuntime()
run_mode = TRACK_MODE_SEEK_LINE
```

行为：

```text
RAW = 0000000:
  左右轮都以 search_speed 低速直行

RAW != 0000000:
  记录当前 error
  切换到 FOLLOW_LINE
```

默认：

```text
search_speed = 100
```

明天观察：

```text
没扫到线时 OLED 显示 M:SEEK
左右轮速度接近 100
扫到线后 OLED 变成 M:LINE
```

## 2. FOLLOW_LINE：正常 PD 循迹

进入条件：

```text
SEEK_LINE 扫到黑线
LOST_RECOVER 找回黑线
L90/R90 转弯后中间传感器重新扫线
```

PD 公式：

```c
correction =
    (track_kp * error + track_kd * (error - last_error)) / track_scale;
```

左右轮输出：

```c
left_speed  = base_speed + correction;
right_speed = base_speed - correction;
```

方向含义：

```text
黑线在右侧 -> error > 0
left_speed 增大
right_speed 减小
小车向右修正
```

当前默认：

```text
base_speed = 160
track_kp = 100
track_kd = 50
track_scale = 100
max_correction = 160
```

## 3. LOST_RECOVER：丢线恢复

进入条件：

```text
FOLLOW_LINE 中 RAW = 0000000
或 L90/R90 转弯超时
```

行为：

```text
进入 M:LOST
左右交替摆头找线
每 160ms 切换一次方向
最多恢复 lost_recover_max_ms
```

默认：

```text
recover_speed = 100
LOST_SWEEP_INTERVAL_MS = 160
lost_recover_max_ms = 1400
```

恢复成功：

```text
RAW != 0000000
切回 M:LINE
继续 PD 循迹
```

恢复失败：

```text
1400ms 仍未扫到线
进入 CAR_STATE_ERROR
电机停止
OLED 显示 ST:ERR
```

## 4. TURN_LEFT_90 / TURN_RIGHT_90：90°转弯动作

90°检测结果显示在 SENSOR 页：

```text
T:0 = 无 90°弯
T:1 = 左 90°弯
T:2 = 右 90°弯
```

检测规则：

```text
左 90°:
  S1/S2 有线
  S6/S7 无线
  error <= -200

右 90°:
  S6/S7 有线
  S1/S2 无线
  error >= 200
```

控制行为：

```text
M:L90:
  左轮 = 0
  右轮 = turn_speed

M:R90:
  左轮 = turn_speed
  右轮 = 0
```

默认：

```text
turn_speed = 130
turn_min_ms = 160
turn_max_ms = 1400
```

退出条件：

```text
已经转动至少 turn_min_ms
并且 S3/S4/S5 中间传感器重新扫到线
切回 M:LINE
```

保护条件：

```text
转动超过 turn_max_ms 仍未扫回中线
切到 M:LOST
```

# 八、明天详细测试步骤

## Test 1：编译与上电

步骤：

```text
1. CCS Clean Project
2. Build Project
3. 烧录
4. 上电
```

预期：

```text
0 Error
OLED 正常显示
默认 ST:READY
电机不转
```

异常：

```text
OLED 不亮：检查 OLED 地址、SDA/SCL、供电、上拉
电机上电就转：立即断电，检查状态机和 TB6612 接线
```

## Test 2：七路 RAW 测试

切到 SENSOR 页，逐个遮挡传感器。

预期：

```text
白底无黑线: 0000000
中间 S4:     0001000
左侧 S1/S2:  0000011
右侧 S6/S7:  1100000
```

重点观察：

```text
RAW 是否稳定
CNT 是否与黑线数量一致
E 是否左负右正
```

异常判断：

```text
黑白反了：修改 TRACK_BLACK_LEVEL
左右反了：检查传感器安装方向或 bit 映射
RAW 抖动大：检查供电、地线、传感器高度
```

## Test 3：启动找线 SEEK_LINE 架空测试

前置：

```text
车轮架空
传感器下方无黑线
RAW = 0000000
```

步骤：

```text
短按 K2
```

预期：

```text
ST:RUN
M:SEEK
L≈100
R≈100
左右轮同向低速转
```

然后把黑线放到传感器下方。

预期：

```text
M:SEEK -> M:LINE
进入正常 PD
```

异常：

```text
RAW=0000000 但不动：检查 search_speed、Motor_SetSpeed、TB6612 STBY
扫到线后仍 M:SEEK：检查 RAW 是否真的非 0
```

## Test 4：正常 PD 循迹方向架空测试

前置：

```text
车轮架空
M:LINE
```

手动让黑线偏右。

预期：

```text
E > 0
L > R
左轮更快，右轮更慢
```

手动让黑线偏左。

预期：

```text
E < 0
L < R
左轮更慢，右轮更快
```

异常：

```text
方向反了：修改 TRACK_REVERSE_CORRECTION 或检查左右电机接线
单侧电机反转：修改 MOTOR_A/B_INVERT_DIRECTION 或交换电机线
```

## Test 5：LOST_RECOVER 架空测试

前置：

```text
先让小车进入 M:LINE
然后移走黑线，使 RAW=0000000
```

预期：

```text
M:LINE -> M:LOST
左右轮单轮交替动作
每约 160ms 换方向
1400ms 内重新给黑线 -> M:LINE
1400ms 仍没线 -> ST:ERR
```

重点观察：

```text
g_appRuntime.lost_elapsed_ms 是否每 20ms 增加
g_appRuntime.lost_count 是否增加
M:LOST 是否能回 M:LINE
```

异常：

```text
一丢线就 ERR：检查 lost_recover_max_ms 是否为 1400
一直 LOST 不回 LINE：检查 RAW 是否恢复非 0
摆头太慢：提高 recover_speed
摆头太猛：降低 recover_speed
```

## Test 6：90°弯检测测试

先不要急着落地高速跑，建议手动用黑线模拟。

左 90°模拟：

```text
让 S1/S2 检测到黑线
S6/S7 不检测
观察 T
```

预期：

```text
T:1
```

右 90°模拟：

```text
让 S6/S7 检测到黑线
S1/S2 不检测
观察 T
```

预期：

```text
T:2
```

异常：

```text
明明是左弯却 T:2：左右 bit 或安装方向反
普通偏线也频繁 T:1/T:2：检测阈值太敏感
90°弯不出 T:1/T:2：阈值太严格或边缘传感器没扫到
```

## Test 7：90°转弯动作架空测试

前置：

```text
车轮架空
M:LINE
手动制造 T:1 或 T:2
```

左 90°预期：

```text
T:1
M:L90
左轮 L=0
右轮 R≈130
至少持续 160ms
中间 S3/S4/S5 扫到线后回 M:LINE
```

右 90°预期：

```text
T:2
M:R90
左轮 L≈130
右轮 R=0
至少持续 160ms
中间 S3/S4/S5 扫到线后回 M:LINE
```

超时保护预期：

```text
如果 1400ms 内中间没扫到线
M:L90/R90 -> M:LOST
```

异常：

```text
转弯方向反了：检查左右轮接线或 L90/R90 动作
刚进入 L90/R90 又立刻回 LINE：turn_min_ms 太小或中心传感器误触
一直 L90/R90 不退出：中心传感器没扫到线或 turn_max_ms 太大
转不过去：提高 turn_speed 或 turn_max_ms
转过头：降低 turn_speed 或 turn_min_ms
```

## Test 8：低速落地直线测试

前置：

```text
前面架空测试通过
人员准备 K3 急停
base_speed = 160
KP = 100
KD = 50
```

步骤：

```text

```

预期：

```text

车身轻微修正

```

异常与调整：

```text

```

## Test 9：低速落地 90°弯测试

前置：

```text
直线稳定
90°架空动作正确
人员准备 K3 急停
```

观察顺序：

```text
接近弯道前：M:LINE
检测急弯时：T:1 或 T:2
执行转弯时：M:L90 或 M:R90
扫回中线后：M:LINE
失败时：M:LOST 或 ST:ERR
```

现象判断：

```text
没进 L90/R90：检测阈值或传感器边缘没扫到
进了 L90/R90 但转不过：turn_speed 太低或 turn_max_ms 太短
进了 L90/R90 后转过头：turn_speed 太高或 turn_min_ms 太长
转弯中掉线后能找回：LOST_RECOVER 正常
转弯后 ERR：turn_max_ms 或 lost_recover_max_ms 不合适
```

# 九、明天参数调整建议

优先级：

```text
先保证 RAW 正确
再保证电机方向正确
再调 PD
最后调转弯参数
```

## 1. 直线参数

当前推荐：

```text
base_speed = 160
track_kp = 100
track_kd = 50
max_correction = 160
```

现象：

```text
蛇形摆动：KP 降到 80~90，或 KD 提到 60
转向不够：KP 提到 110~130
速度太快：base_speed 降到 130~150
速度太慢：base_speed 提到 180
```

## 2. 找线参数

```text
search_speed = 100
recover_speed = 100
lost_recover_max_ms = 1400
```

现象：

```text
SEEK_LINE 找线太慢：提高 search_speed
LOST_RECOVER 摆头太慢：提高 recover_speed
容易冲出去：降低 recover_speed 或缩短 lost_recover_max_ms
恢复时间不够：继续提高 lost_recover_max_ms
```

## 3. 90°转弯参数

```text
turn_speed = 130
turn_min_ms = 160
turn_max_ms = 1400
```

现象：

```text
转不过去：turn_speed 提到 150~170
转弯动作太短：turn_min_ms 提到 200
转过头：turn_speed 降到 110~120
迟迟不退出 L90/R90：检查中心传感器，或降低 turn_min_ms
经常转弯超时进 LOST：继续提高 turn_max_ms
```

# 十、当前风险点

```text
1. 90°检测阈值尚未实车验证，可能误触发。
2. L90/R90 使用单轮转弯，窄弯可能转不过。
3. turn_speed / turn_min_ms / turn_max_ms 暂未加入菜单，需要改代码默认值。
4. LOST_RECOVER 会让车继续动作 1400ms，测试时必须有人准备 K3 急停。
5. 当前无计圈，target_laps 仍不会触发自动停车。
6. 编码器未参与速度闭环，左右电机机械差异仍可能导致偏航。
```

# 十二、结论

当前最新工程已经从基础循迹升级为带子模式的巡线控制框架：

```text
SEEK_LINE：启动找线
FOLLOW_LINE：正常 PD 循迹
LOST_RECOVER：丢线左右摆头恢复
TURN_LEFT_90：左 90°单轮转弯
TURN_RIGHT_90：右 90°单轮转弯
```

明天测试重点不是一次性追求跑完整赛道，而是按顺序确认：

```text
RAW 正确
电机方向正确
SEEK 正常
LINE 稳定
LOST 能恢复
T 检测准确
L90/R90 动作正确
最后再低速落地跑完整线路
```
