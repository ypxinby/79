# 二维云台系统设计文档

> 文档状态：P23V 实机验收基线  
> 基线提交：`5833450 P23V PITCH 追踪改为连续速度模式`  
> 适用工程：MSPM0G3507 + D36A 双路步进电机云台  
> 更新日期：2026-07-18

## 1. 文档目的

本文档统一说明当前二维云台的：

1. 硬件连接与底层 STEP/DIR/EN 设计；
2. YAW、PITCH 两轴的位置和速度控制设计；
3. 调度周期、状态反馈和软件限位；
4. 对外接口及其使用边界；
5. 当前 Tracker 的误差处理链路；
6. 后续接入 K230 视觉追踪的推荐架构；
7. 从当前版本到完整双轴追踪需要执行的步骤和验收标准。

《云台调试记录.md》继续用于记录每次实机问题和修订时间线；《云台运控逻辑.md》保留早期总体方案。若两者与本文档冲突，以当前代码和本文档为准。

## 2. 当前已验收结论

截至 P23V，以下功能已通过实机验证：

- YAW 第一电机 STEP/DIR/EN 正常；
- YAW 相对位置、最短路径目标、RPM/加速度限制和世界锁定正常；
- PITCH 第二电机 PB5/PB6/PB7 基础输出正常；
- PITCH 在 GPIT P21 页面能够重复执行正反向 ±10°位置运动；
- PITCH 上电相对零点和 ±50°软件限位正常；
- PITCH 在 GTRK P23V 页面能够连续正转、反转和快速换向；
- PITCH 连续按 K3、K2/K3 快速交替均不会再出现单方向锁死；
- P21 位置模式和 P23V 追踪速度模式可以共存。

P23V 的关键结论是：

```text
PITCH 手动动作 -> 使用位置模式
PITCH 视觉追踪 -> 使用连续速度模式
```

视觉追踪不能再通过“每 10ms 增加约一两个微步的位置目标”驱动 PITCH。该方式会造成频繁启停，并容易受到重力、摩擦和机械间隙影响。

## 3. 系统总体架构

### 3.1 控制链路

```text
K230/按键模拟目标
        ↓
GimbalTracker：误差滤波、死区、P 控制和限速
        ↓
┌──────────────────┬────────────────────┐
│ YAW wrapped 位置目标 │ PITCH 有符号追踪速度 │
└──────────────────┴────────────────────┘
        ↓                         ↓
YAW 位置/RPM控制             PITCH 位置/速度双模式
        ↓                         ↓
STEP 半周期、DIR、EN
        ↓
100us SysTick 输出 STEP 并累计有效上升沿
```

云台链路独立于小车任务链，不经过：

```text
MissionManager -> MotionAction -> CarController -> Motor
```

后续任务层只能通过公开的 `Gimbal_*` 接口控制云台，不能直接操作 PB4、PB5、PB6、PB7、PA1、PA2。

### 3.2 主要源码

| 文件 | 职责 |
| --- | --- |
| `gimbal.c/.h` | 双轴业务控制、PITCH 底层、位置/速度模式、限位和反馈 |
| `gimbal_stepper.c/.h` | YAW STEP/DIR/EN 底层与有效步数累计 |
| `gimbal_tracker.c/.h` | 视觉误差滤波、死区、速度计算、目标有效性管理 |
| `empty.c` | 100us、5ms、10ms 调度 |
| `menu.c` | GYAW、GPIT、GTRK 测试页按键输入 |
| `oled_ui.c` | 云台状态和调试反馈显示 |
| `empty.syscfg` | 云台 GPIO 配置来源 |

当前 YAW 已拆出独立 `gimbal_stepper`，PITCH STEP 实现仍在 `gimbal.c` 内。后续若需要继续扩展两轴统一驱动，可将 PITCH 也迁移到通用 axis/stepper 模块，但这不是接入 K230 前的阻塞项。

## 4. 硬件与基础参数

### 4.1 引脚连接

| 轴 | 信号 | MSPM0 引脚 | D36A |
| --- | --- | --- | --- |
| YAW | STEP | PB4 | ST1 |
| YAW | DIR | PA1 | DIR1 |
| YAW | EN | PA2 | EN1 |
| PITCH | STEP | PB5 | ST2 |
| PITCH | DIR | PB6 | DIR2 |
| PITCH | EN | PB7 | EN2 |

当前 D36A 使能逻辑按高电平工作处理：

```c
GIMBAL_EN_ACTIVE_LOW = 0
GIMBAL_PITCH_EN_ACTIVE_LOW = 0
```

方向极性分别由底层宏控制。视觉方向不正确时，应优先修改 Tracker 的方向映射，不应因为画面方向相反就修改已经通过手动验收的底层 DIR 极性。

### 4.2 步数与周期

当前两轴均按以下参数计算：

```text
每圈有效 STEP：3200
每个有效 STEP：360° / 3200 = 0.1125°
STEP 调度周期：100us
STEP 调度频率：10kHz
控制周期：5ms
Tracker 更新周期：10ms
```

只有 STEP 上升沿计为一个有效步数：

```c
estimated_steps += direction;
```

GPIO 高低翻转次数不能直接当作步数。

## 5. 调度设计

| 周期 | 调用 | 主要职责 |
| --- | --- | --- |
| 100us | `Gimbal_Tick100us()` | 产生 STEP 边沿、累计有效步数、检查位置完成或软件边界 |
| 5ms | `Gimbal_Update5ms()` | RPM 斜坡、方向与 STEP 半周期更新、反馈刷新 |
| 10ms | `GimbalTracker_Update(0.010f)` | 误差滤波、死区、P 控制、YAW/PITCH 追踪命令 |
| 20ms | `App_Update_20ms()` | 按键、菜单、OLED 和整车业务 |

100us 中断只承担高频且确定的工作，不应加入：

- 浮点视觉控制；
- UART 协议解析；
- OLED 输出；
- 复杂任务状态机；
- 阻塞等待或日志打印。

PITCH 追踪速度模式切换方向和 STEP 状态时使用短临界区，避免 100us 中断在状态更新中间产生 STEP。

## 6. 位置与反馈模型

### 6.1 开环位置估算

两轴当前位置来自已输出 STEP 的软件累计：

```text
estimated_steps -> continuous_deg_x10 -> wrapped_deg_x10
```

这是开环估算，不是编码器真实反馈。因此：

- 软件能够知道自己输出了多少脉冲；
- 软件不能直接检测电机失步；
- 机械卡死、驱动掉电或扭矩不足时，软件位置可能与物理位置不一致。

### 6.2 YAW 位置模型

YAW 支持：

- 连续累计角度；
- `0~360°` 单圈角度；
- 最短路径 wrapped 目标；
- 累计圈数；
- IMU 世界锁定补偿。

YAW 当前不使用普通 ±角度软限位。正式装机前必须确认线缆是否允许连续旋转；没有滑环时需要增加圈数或线缆限位。

### 6.3 PITCH 零点与限位

PITCH 使用上电相对零点：

```text
上电前将俯仰轴放到机械中间
Gimbal_PitchInit() 将当前位置定义为 0°
```

当前软件限位：

```text
最小角度：-50.0°
最大角度：+50.0°
```

所有 PITCH 位置接口和追踪速度模式都必须受该限位约束：

```text
到达 +50°：禁止继续正向，允许负向返回
到达 -50°：禁止继续负向，允许正向返回
```

### 6.4 反馈结构

`GimbalFeedback` 中最重要的字段：

| 字段 | 含义 |
| --- | --- |
| `estimated_steps` | 当前累计有效步数 |
| `continuous_deg_x10` | 当前连续角度 ×10 |
| `wrapped_deg_x10` | 当前单圈角度 ×10 |
| `target_deg_x10` | 位置模式目标角度 ×10 |
| `target_rpm_x10` | 当前目标 RPM ×10 |
| `commanded_rpm_x10` | 经过斜坡后的实际命令 RPM ×10 |
| `step_half_period_ticks` | STEP 半周期 tick 数 |
| `direction` | 当前方向，`+1/-1` |
| `running` | STEP 状态机是否运行 |
| `enabled` | 驱动器是否使能 |
| `limit_direction` | `1/-1/0`，表示正限位、负限位或无限幅 |
| `positive_limit` | 是否到达正边界 |
| `negative_limit` | 是否到达负边界 |
| `position_valid` | 软件位置是否有效 |

## 7. YAW 底层设计

YAW 由 `gimbal.c` 控制层和 `gimbal_stepper.c` STEP 层组成。

### 7.1 位置控制

YAW 位置命令最终转换为连续目标步数。wrapped 目标先计算 `[-180°, +180°]` 最短误差，再映射到距离当前位置最近的连续目标。

### 7.2 速度与加速度

当前参数：

```text
普通位置最大速度：4 RPM
普通位置加速度：10 RPM/s
世界锁定最大速度：12 RPM
世界锁定加速度：30 RPM/s
```

5ms 控制层将目标 RPM 经过加速度限制后换算成 STEP 半周期。

### 7.3 世界锁定

世界锁定记录启用时的世界方向：

```text
locked_world_yaw = car_yaw + gimbal_relative_yaw
```

运行时根据 IMU 车体角度持续更新云台目标：

```text
gimbal_target = locked_world_yaw - current_car_yaw
```

## 8. PITCH 底层设计

PITCH 具有两个独立控制模式。

### 8.1 位置模式

位置模式用于：

- GPIT P21 手动 ±10°；
- 指定绝对角度；
- 指定绝对步数；
- 任务层固定姿态动作；
- 零点和软件限位验收。

位置命令确定最终目标步数，100us STEP 层在达到目标后停止并保持。

### 8.2 追踪速度模式

追踪速度模式用于：

- P23V 按键模拟 error_y；
- 后续 K230 连续视觉追踪；
- 需要持续正反向修正的控制场景。

接口直接接收有符号角速度：

```c
Gimbal_PitchSetTrackingSpeedDegS(speed_deg_s);
```

角速度转换为 RPM：

```text
RPM = deg/s / 6
```

当前最大值受 `4 RPM` 限制，即最大约 `24°/s`。

速度模式不设置“一两个步之外”的微小目标，而是持续输出 STEP，直到：

- Tracker 下发零速度；
- Observation 失效；
- Tracker 被禁用；
- 达到 PITCH 软件限位；
- 上层执行 StopHold 或 Release。

### 8.3 换向处理

P23V 当前采用确定性换向：

```text
停止 STEP并拉低
清零旧方向 target/commanded RPM
切换 DIR
从 0 向新方向重新加速
```

该方式已经通过 K2/K3 快速交替实机验收。后续如果机械冲击明显，可以改成“先减速到零，再切 DIR”的平滑状态机，但不能重新引入旧方向 RPM 与新 DIR 同时存在的状态。

### 8.4 模式切换规则

```text
位置接口被调用 -> 切换为 POSITION
追踪速度接口被调用 -> 切换为 TRACK_SPEED
StopHold/Release/ConfirmZero -> 清除追踪速度状态
```

手动位置模式和视觉速度模式不能同时控制 PITCH。

## 9. 对外接口设计

### 9.1 初始化与调度接口

| 接口 | 调用位置 | 说明 |
| --- | --- | --- |
| `Gimbal_Init()` | `App_Init()` | 初始化 YAW 和 PITCH |
| `Gimbal_Tick100us()` | SysTick | 双轴 STEP 高频调度 |
| `Gimbal_Update5ms()` | 主循环 pending | 双轴控制更新 |

业务层优先使用统一初始化和调度接口，不应分别重复调用单轴初始化。

### 9.2 YAW 接口

| 接口 | 用途 |
| --- | --- |
| `Gimbal_YawMoveToDeg()` | 连续坐标绝对角度位置命令 |
| `Gimbal_YawMoveWrappedDeg()` | 单圈角度最短路径运动，重置速度斜坡 |
| `Gimbal_YawSetWrappedTargetDeg()` | 实时刷新 wrapped 目标，不主动重置速度斜坡 |
| `Gimbal_YawMoveRelativeDeg()` | 相对角度运动 |
| `Gimbal_YawEnableWorldLock()` | 锁定当前世界方向 |
| `Gimbal_YawDisableWorldLock()` | 退出世界锁定并保持 |
| `Gimbal_YawStopHold()` | 停止并保持使能 |
| `Gimbal_YawRelease()` | 停止并释放驱动器 |
| `Gimbal_YawGetFeedback()` | 获取 YAW 状态反馈 |

### 9.3 PITCH 接口

| 接口 | 模式 | 用途 |
| --- | --- | --- |
| `Gimbal_PitchMoveToDeg()` | 位置 | 运动到绝对角度 |
| `Gimbal_PitchSetTargetDeg()` | 位置 | 实时刷新绝对位置目标 |
| `Gimbal_PitchMoveRelativeDeg()` | 位置 | 从当前实际位置运动相对角度 |
| `Gimbal_PitchMoveToSteps()` | 位置 | 运动到绝对步数 |
| `Gimbal_PitchSetTargetSteps()` | 位置 | 实时刷新绝对目标步数 |
| `Gimbal_PitchMoveRelativeSteps()` | 位置 | 相对步数运动 |
| `Gimbal_PitchSetTrackingSpeedDegS()` | 速度 | 视觉追踪有符号角速度命令 |
| `Gimbal_PitchStopTrackingHold()` | 停止 | 退出追踪并保持 |
| `Gimbal_PitchConfirmZero()` | 零点 | 停止状态下将当前位置定义为 0° |
| `Gimbal_PitchStopHold()` | 停止 | 停止、保持并清除速度模式 |
| `Gimbal_PitchRelease()` | 释放 | 停止、释放并清除速度模式 |
| `Gimbal_PitchGetFeedback()` | 反馈 | 获取 PITCH 状态 |

使用规则：

```text
固定姿态/任务动作：使用位置接口
视觉误差连续修正：使用 TrackingSpeed 接口
禁止 Tracker 再用 SetTargetDeg 模拟速度
```

### 9.4 Tracker 接口

| 接口 | 用途 |
| --- | --- |
| `GimbalTracker_Init()` | 初始化 Tracker 状态 |
| `GimbalTracker_Enable()` | 启用或禁用追踪 |
| `GimbalTracker_PushObservation()` | 提交最新视觉观测 |
| `GimbalTracker_ClearObservation()` | 目标失效并停止追踪 |
| `GimbalTracker_Update(dt_s)` | 10ms 固定周期控制更新 |
| `GimbalTracker_GetFeedback()` | OLED 和调试状态读取 |

Observation 当前字段：

```c
typedef struct {
    int16_t error_x_px;
    int16_t error_y_px;
    uint8_t valid;
    uint16_t sequence;
    uint32_t timestamp_ms;
} GimbalTargetObservation;
```

## 10. 当前 Tracker 设计

### 10.1 参数

```text
更新周期：10ms
滤波系数 alpha：0.35
像素死区：8px
YAW Kp：0.08 deg/s/px
PITCH Kp：0.06 deg/s/px
YAW 最大速度：30 deg/s
PITCH 最大速度：20 deg/s
```

方向映射：

```c
GIMBAL_TRACKER_YAW_ERROR_SIGN
GIMBAL_TRACKER_PITCH_ERROR_SIGN
```

### 10.2 数据处理

```text
最新 Observation
    ↓
一阶低通滤波
    ↓
像素死区
    ↓
Kp × error 得到目标角速度
    ↓
速度限幅
    ↓
YAW：积分成 wrapped 目标
PITCH：直接进入连续速度模式
```

YAW 当前保持已经实测稳定的目标积分方式；PITCH 使用 P23V 连续速度模式。后续若需要完全统一两轴追踪行为，可在 YAW 上增加对应速度接口，但应作为独立阶段验证，不能同时重写两轴。

### 10.3 模拟 Observation

P23X/P23V 按键模拟输入使用：

```text
timestamp_ms = 0
有效时间约 500ms
每次按键重新同步当前位置并重置滤波起点
```

真实 K230 数据必须携带非零时间戳或由 MSPM0 在接收时补充本地时间，并持续刷新。

### 10.4 无效目标行为

以下情况均应停止追踪并保持：

- Tracker Disable；
- `valid = 0`；
- 模拟 Observation 超时；
- 后续 K230 通信超时；
- 协议或 CRC 校验失败且超过有效期。

## 11. K230 追踪预计设计

### 11.1 职责划分

K230 负责：

- 摄像头采集和模型推理；
- 检测框与目标中心计算；
- 多目标选择；
- 置信度和目标有效性判断；
- 发送序列号、时间戳和目标数据。

MSPM0 负责：

- UART 接收和协议解析；
- CRC、版本和字段范围校验；
- 将协议数据转换成 `GimbalTargetObservation`；
- 误差滤波、死区和速度控制；
- 软件限位、STEP/DIR/EN 和超时保护。

K230 不直接控制 STEP/DIR/EN。

### 11.2 推荐协议字段

建议协议至少包含：

```text
帧头
协议版本
payload长度
sequence
K230 timestamp_ms
valid
frame_width / frame_height
target_center_x / target_center_y
confidence
可选：bbox宽高、目标ID
CRC16
```

MSPM0 接收后计算：

```text
error_x = target_center_x - frame_width / 2
error_y = target_center_y - frame_height / 2
```

如果 K230 直接发送 `error_x/error_y`，仍建议保留画面宽高，以便后续改成归一化误差和适配不同分辨率。

### 11.3 接收层结构

推荐新增独立模块：

```text
k230_protocol.c/.h       帧格式、CRC和字段解析
k230_receiver.c/.h       UART字节流、缓冲区和超时
gimbal_vision_adapter.c  协议目标 -> GimbalTargetObservation
```

协议层不能直接调用 GPIO，也不应直接修改 PITCH/YAW 内部状态。

### 11.4 时间有效性

建议状态：

| 状态 | 条件 | 行为 |
| --- | --- | --- |
| TRACKING | `valid=1` 且数据新鲜 | 正常输出双轴追踪速度 |
| LOST_SHORT | 短暂丢失目标 | 目标速度衰减到 0 并保持 |
| LOST | 连续无目标 | StopHold，可选择进入搜索 |
| COMM_TIMEOUT | UART 数据超时 | 立即退出追踪并保持 |
| PROTOCOL_ERROR | CRC/版本/范围非法 | 丢弃帧，不刷新有效时间 |

初始建议：

```text
目标数据新鲜期：100ms
短暂丢失窗口：100~300ms
通信超时：300ms
```

### 11.5 最终双轴输出

第一版接 K230 时保留当前混合输出：

```text
YAW：wrapped 目标积分
PITCH：连续速度模式
```

该方案改动小，已经分别通过 P23X 和 P23V 验证。

后续最终形态建议两轴都提供追踪速度模式：

```text
error_x -> yaw_speed_deg_s -> YAW tracking speed
error_y -> pitch_speed_deg_s -> PITCH tracking speed
```

迁移 YAW 时必须保留世界锁定和 wrapped 位置接口，不得为了视觉追踪破坏现有位置功能。

### 11.6 目标搜索规划

目标丢失后的搜索不应在第一版 K230 接入时立即实现。推荐顺序：

1. 先停止并保持；
2. 确认通信和目标恢复稳定；
3. 再增加低速 YAW 扫描；
4. 必要时增加 PITCH 小范围扫描；
5. 重新发现目标后平滑切回 TRACKING。

## 12. 后续执行步骤

以下步骤按顺序执行，不建议跨阶段同时修改。

### 步骤 1：冻结当前云台基线

基线：

```text
5833450 P23V PITCH 追踪改为连续速度模式
```

验收：P21、P23X、P23V 全部正常，代码和文档已推送 GitHub。

### 步骤 2：定义 K230 通信协议

输出协议说明，包括字段、字节序、长度、序列号、时间戳、CRC 和错误处理。

验收：使用电脑脚本能够生成合法帧和错误帧。

### 步骤 3：实现 MSPM0 UART 接收层

实现非阻塞接收、环形缓冲或帧缓冲，不在中断中解析复杂协议。

验收：连续接收数据不阻塞 100us STEP 调度和 20ms 小车任务。

### 步骤 4：实现协议解析与 CRC

完成帧头同步、长度检查、CRC、版本和字段范围检查。

验收：合法帧通过；截断、错位、错误 CRC 和超范围字段被丢弃。

### 步骤 5：实现视觉适配层

将协议目标转换为 `GimbalTargetObservation`，统一 sequence、timestamp、valid 和 error_x/error_y。

验收：不连接电机时，OLED 能正确显示真实 K230 误差和数据有效状态。

### 步骤 6：实现数据新鲜度与通信超时

增加本地接收时间、目标年龄和通信状态。

验收：拔掉 K230 或停止发帧后，云台在规定时间内 StopHold，不继续使用旧误差。

### 步骤 7：接入真实 PITCH 单轴追踪

暂时将 YAW error_x 置零，只让真实 error_y 驱动 PITCH 速度模式。

验收：目标上下移动时 PITCH 双方向连续修正；目标居中后停止；±50°限位有效。

### 步骤 8：接入真实 YAW 单轴追踪

暂时将 PITCH error_y 置零，验证 YAW 方向、滤波、死区和 wrapped 目标刷新。

验收：目标左右移动时 YAW 双方向修正，不影响 PITCH 保持。

### 步骤 9：双轴同时追踪

同时启用 error_x/error_y，验证调度负载和两轴互不覆盖。

验收：目标斜向移动时两轴同步响应；OLED、小车任务和 STEP 输出无明显卡顿。

### 步骤 10：参数标定

依次调整：

```text
方向符号
像素或归一化死区
滤波 alpha
YAW/PITCH Kp
最大追踪速度
加速度限制
目标丢失时间
```

一次只调整一类参数并记录结果。

### 步骤 11：目标丢失与恢复

实现 LOST_SHORT、LOST、COMM_TIMEOUT 状态，并验证重新出现目标时不会突然跳转。

### 步骤 12：搜索模式

在停止保持策略稳定后，再增加低速搜索和重新捕获逻辑。

### 步骤 13：任务层集成

由 Mission/Action 层控制 Tracker Enable、期望姿态和搜索状态，但任务层不能直接控制 STEP 或修改 Tracker 内部变量。

### 步骤 14：长期可靠性测试

至少验证：

- 连续追踪 10~30 分钟；
- K230 反复断连重连；
- 目标快速跨越画面中心；
- PITCH 正负限位反复进入和退出；
- 小车运动、IMU、超声波、OLED 与云台同时运行；
- 电机无持续顶限位、异常发热和明显丢步。

## 13. 调试页面与验收读数

### GPIT P21

用于 PITCH 位置模式、零点和 ±50°限位验收。

### GTRK P23X

用于 YAW 模拟 error_x 追踪验收。

### GTRK P23V

用于 PITCH 连续速度追踪验收：

```text
K2：error_y = +240
K3：error_y = -240
K2长按：Enable/Disable Tracker
K3长按：ClearObservation
```

关键读数：

```text
PS：Tracker PITCH 目标角速度 ×10
R：PITCH commanded RPM ×10
PA：PITCH 软件估算实际角度 ×10
D：PITCH 当前方向 +1/-1
```

正常示例：

```text
K2：PS约+144，R向+24爬升，D=1，PA增加
K3：PS约-144，R向-24爬升，D=-1，PA减小
```

## 14. 开发约束

1. 不在任务层或协议层直接操作 STEP/DIR/EN；
2. PITCH 视觉追踪只使用速度接口；
3. P21 位置接口修改后必须重新做正反向和限位回归；
4. 修改 100us STEP 路径后必须检查中断执行时间；
5. 修改方向映射时优先改 Tracker sign，不改已验收底层极性；
6. 不把 OLED 数值当作真实编码器位置；
7. 每个阶段单独提交，并在《云台调试记录.md》中记录实机结果；
8. 未通过单轴验收前，不进入双轴和任务层集成；
9. 通信超时必须 StopHold，不能无限复用最后一帧；
10. 正式上车前确认 YAW 线缆绕转限制和 PITCH 机械安全范围。
