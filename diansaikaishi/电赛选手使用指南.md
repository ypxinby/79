# 电赛选手使用指南

> 这份文档给准备直接用本工程跑电赛地图的同学看。重点只有一个：在现有动作积木上，快速拼出自己的比赛 MAP。

## 1. 主要改哪些文件

| 文件 | 什么时候改 |
| --- | --- |
| `app_features.h` | 切换比赛模式 / 开发模式 |
| `mission_library.c` | 拼 `MAP-A / MAP-B / MAP-C`，绝大多数地图修改都在这里 |
| `mission_library.h` | 新增 `MAP-D` 这类任务 ID 时才改 |
| `app_config.c` | 需要改默认速度、PID、绕障默认参数时再改 |

## 2. 比赛模式和开发模式

打开 `app_features.h`：

```c
#define APP_PROFILE                APP_PROFILE_COMPETITION
```

比赛前使用：

```c
#define APP_PROFILE                APP_PROFILE_COMPETITION
```

OLED 只显示正式地图：

```text
1/3 MAP-A
2/3 MAP-B
3/3 MAP-C
```

调试时使用：

```c
#define APP_PROFILE                APP_PROFILE_DEVELOPMENT
```

OLED 显示：

```text
1/8 TEST-BASIC
2/8 TEST-R90
3/8 TEST-YH
4/8 TEST-OBS-F
5/8 TEST-STOP
6/8 MAP-A
7/8 MAP-B
8/8 MAP-C
```

改完模式后，需要重新编译烧录。

## 3. 建议先测的测试任务

| 任务 | 用来确认什么 |
| --- | --- |
| `TEST-BASIC` | 找线、循迹、灰度基础是否正常 |
| `TEST-R90` | 右 90 度路口检测和右转是否正常 |
| `TEST-YH` | IMU 相对转角、保持航向直行是否正常 |
| `TEST-STOP` | 前方遇障停车等待是否正常 |
| `TEST-OBS-F` | 固定主动绕障整条链路是否正常 |

这些能跑，再开始拼正式地图。

## 4. OLED 上怎么看任务

STATUS 页：

```text
TASK:1/3 READY
```

表示当前是可见任务列表里的第 1 个，一共有 3 个任务。

PARAM 页调 `TASK` 时：

```text
VAL:1
ID:10 N:3 NX:2
```

| 字段 | 含义 |
| --- | --- |
| `VAL:1` | 当前选第 1 个可见任务 |
| `ID:10` | 内部任务 ID，一般不用管 |
| `N:3` | 当前模式下一共有 3 个可见任务 |
| `NX:2` | 再按一次增加，会到第 2 个可见任务 |

比赛现场主要看 `TASK:x/y` 和 `VAL`。

## 5. MAP 在哪里写

打开 `mission_library.c`，找到：

```c
static const MotionAction g_missionMapA[] = {
    ...
};

static const MotionAction g_missionMapB[] = {
    ...
};

static const MotionAction g_missionMapC[] = {
    ...
};
```

这三个数组对应 OLED 里的 `MAP-A / MAP-B / MAP-C`。

每一行 `ACTION_...` 是一个动作，小车会从上往下执行，最后用：

```c
ACTION_STOP()
```

结束任务。

## 6. 常用动作积木表

| 你想让车做什么 | 写法 |
| --- | --- |
| 按起点 yaw 找第一条线 | `ACTION_SEEK_MISSION_YAW(0.0f, MAP_SEEK_TIMEOUT_MS)` |
| 按当前 yaw 偏移找线 | `ACTION_SEEK_CURRENT_YAW(angle, timeout)` |
| 按 `REV` 角度第二次找线 | `ACTION_SEEK_SECOND_CONFIG(MAP_SEEK_TIMEOUT_MS)` |
| 一直循迹 | `ACTION_FOLLOW_FOREVER(0U)` |
| 循迹固定时间 | `ACTION_FOLLOW_FOR_TIME(duration_ms, timeout)` |
| 循迹到出线 | `ACTION_FOLLOW_UNTIL_LINE_LOST(MAP_FINAL_TIMEOUT_MS)` |
| 循迹到左 90 路口 | `ACTION_FOLLOW_UNTIL_LEFT_90(MAP_FOLLOW_TIMEOUT_MS)` |
| 循迹到右 90 路口 | `ACTION_FOLLOW_UNTIL_RIGHT_90(MAP_FOLLOW_TIMEOUT_MS)` |
| 循迹到任意 90 路口 | `ACTION_FOLLOW_UNTIL_ANY_90(MAP_FOLLOW_TIMEOUT_MS)` |
| 左转 90 度 | `ACTION_TURN_LEFT_90(MAP_TURN_TIMEOUT_MS)` |
| 右转 90 度 | `ACTION_TURN_RIGHT_90(MAP_TURN_TIMEOUT_MS)` |
| 按 IMU 相对转角 | `ACTION_TURN_RELATIVE_YAW(angle, timeout)` |
| 保持航向直行一段时间 | `ACTION_DRIVE_HEADING_TIME(duration_ms, timeout)` |
| 重新找回线 | `ACTION_REACQUIRE_LINE(timeout)` |
| 原地等待 | `ACTION_WAIT_MS(wait_ms)` |
| 停车结束任务 | `ACTION_STOP()` |

最常用的组合是：

```text
SEEK -> FOLLOW 到路口 -> TURN -> FOLLOW 到下一个事件 -> STOP
```

## 7. 拼一个 MAP 的例子

目标路线：

```text
找线 -> 右 90 路口 -> 右转 -> 左 90 路口 -> 左转 -> 出线停车
```

写成：

```c
static const MotionAction g_missionMapA[] = {
    ACTION_SEEK_MISSION_YAW(0.0f, MAP_SEEK_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_RIGHT_90(MAP_FOLLOW_TIMEOUT_MS),
    ACTION_TURN_RIGHT_90(MAP_TURN_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_LEFT_90(MAP_FOLLOW_TIMEOUT_MS),
    ACTION_TURN_LEFT_90(MAP_TURN_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_LINE_LOST(MAP_FINAL_TIMEOUT_MS),
    ACTION_STOP()
};
```

注意：

```text
ACTION_FOLLOW_UNTIL_RIGHT_90 只负责检测右 90，不会自动右转
要右转，下一行必须写 ACTION_TURN_RIGHT_90
```

左 90 同理。

## 8. timeout 怎么写

当前默认值在 `mission_library.c`：

| 名称 | 默认值 | 用途 |
| --- | ---: | --- |
| `MAP_SEEK_TIMEOUT_MS` | `4000U` | 找线最多 4 秒 |
| `MAP_FOLLOW_TIMEOUT_MS` | `8000U` | 等路口最多 8 秒 |
| `MAP_TURN_TIMEOUT_MS` | `2500U` | 转弯最多 2.5 秒 |
| `MAP_FINAL_TIMEOUT_MS` | `5000U` | 等出线最多 5 秒 |

如果某一段特别长，可以单独写具体值：

```c
ACTION_FOLLOW_UNTIL_RIGHT_90(12000U)
```

正式地图里尽量不要写：

```c
0U
```

`0U` 表示永不超时。找不到线或路口时，小车可能一直卡住。

## 9. 障碍物怎么处理

默认 `ACTION_FOLLOW_*` 是固定主动绕障，方向默认右绕。

固定主动绕障大致效果：

```text
遇障 -> 停车等待 -> 转出 -> 直行绕开 -> 转回 -> 找回线 -> 继续原 FOLLOW
```

如果某一段只想遇障停车等待，不主动绕障：

```c
ACTION_FOLLOW_FOREVER_WITH_OBSTACLE(
    OBSTACLE_POLICY_STOP_ONLY,
    BYPASS_DIRECTION_RIGHT,
    0U)
```

如果某一段要指定左绕：

```c
ACTION_FOLLOW_UNTIL_RIGHT_90_WITH_OBSTACLE(
    OBSTACLE_POLICY_FIXED_BYPASS,
    BYPASS_DIRECTION_LEFT,
    MAP_FOLLOW_TIMEOUT_MS)
```

如果要右绕：

```c
BYPASS_DIRECTION_RIGHT
```

## 10. 如何新增 MAP-D

第一步，在 `mission_library.h` 增加 ID：

```c
#define MISSION_ID_MAP_D                (13U)
```

第二步，在 `mission_library.c` 新增动作数组：

```c
static const MotionAction g_missionMapD[] = {
    ACTION_SEEK_MISSION_YAW(0.0f, MAP_SEEK_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_RIGHT_90(MAP_FOLLOW_TIMEOUT_MS),
    ACTION_TURN_RIGHT_90(MAP_TURN_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_LINE_LOST(MAP_FINAL_TIMEOUT_MS),
    ACTION_STOP()
};
```

第三步，在 `g_missionRegistry[]` 里注册：

```c
{
    .mission_id = MISSION_ID_MAP_D,
    .name = "MAP-D",
    .actions = g_missionMapD,
    .action_count = ARRAY_SIZE(g_missionMapD),
    .control_profile_id = 0U
}
```

如果它不是最后一个条目，前一个条目后面要加逗号。

## 11. OLED 常用调参

| 参数 | 含义 | 常见调整 |
| --- | --- | --- |
| `TASK` | 选择任务 | 切 `MAP-A/B/C` |
| `SPD` | 基础速度 | 太慢就加，太飘就降 |
| `YAW` | 第一次找线方向微调 | 找线偏左/偏右时调 |
| `REV` | 第二次找线角度 | 第二次找线方向不准时调 |
| `KP` | 循迹比例 | 弯道跟不上可适当加 |
| `KD` | 循迹微分 | 左右晃动明显可调 |
| `MAX` | 最大循迹修正 | 弯道修正不够可加 |
| `SV` | 舵机前向角度 | 超声波不正对前方时调 |

绕障默认参数在 `app_config.c`：

| 参数 | 含义 |
| --- | --- |
| `avoid_turn_out_deg` | 绕障转出角度 |
| `avoid_drive_out_ms` | 绕障直行时间 |
| `avoid_turn_to_line_deg` | 绕障转回找线角度 |
| `avoid_wait_before_ms` | 遇障后原地等待时间 |
| `avoid_reacquire_settle_ms` | 回线稳定时间 |

## 12. 推荐拼地图流程

不要一次写完整地图。建议每次只加一小段。

| 步骤 | 做法 |
| --- | --- |
| 1 | 写：找线 -> 第一处路口 -> 停车 |
| 2 | 确认路口能识别 |
| 3 | 加第一处转弯 |
| 4 | 加下一段 FOLLOW |
| 5 | 每加一段就编译烧录测试 |
| 6 | 最后再调 timeout 和障碍策略 |

第一版可以先写：

```c
static const MotionAction g_missionMapA[] = {
    ACTION_SEEK_MISSION_YAW(0.0f, MAP_SEEK_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_RIGHT_90(MAP_FOLLOW_TIMEOUT_MS),
    ACTION_STOP()
};
```

确认右 90 能识别后，再改成：

```c
static const MotionAction g_missionMapA[] = {
    ACTION_SEEK_MISSION_YAW(0.0f, MAP_SEEK_TIMEOUT_MS),
    ACTION_FOLLOW_UNTIL_RIGHT_90(MAP_FOLLOW_TIMEOUT_MS),
    ACTION_TURN_RIGHT_90(MAP_TURN_TIMEOUT_MS),
    ACTION_STOP()
};
```

这样最容易定位问题。

## 13. 常见坑

| 坑 | 正确做法 |
| --- | --- |
| `FOLLOW_UNTIL_RIGHT_90` 后车没有右转 | 后面接 `ACTION_TURN_RIGHT_90(...)` |
| `FOLLOW_UNTIL_LEFT_90` 后车没有左转 | 后面接 `ACTION_TURN_LEFT_90(...)` |
| 正式地图用了 `0U` 超时 | 改成合理 timeout |
| 比赛现场出现测试任务 | 确认 `APP_PROFILE_COMPETITION` |
| OLED 显示 `C/L/R/SC/D` 扫描字段 | 确认 `FEATURE_OBSTACLE_SCANNER` 是 `0` |
| 任务跑完没有停 | 最后一行确认有 `ACTION_STOP()` |

## 14. 赛前检查清单

| 检查项 | 应该是什么 |
| --- | --- |
| `APP_PROFILE` | `APP_PROFILE_COMPETITION` |
| `FEATURE_OBSTACLE_SCANNER` | `0` |
| OLED 任务列表 | 只显示正式 MAP |
| 每个正式 MAP 结尾 | `ACTION_STOP()` |
| 正式 MAP 超时 | SEEK / TURN / 有结束条件的 FOLLOW 都有非 0 timeout |
| 基础测试 | `TEST-BASIC / TEST-R90 / TEST-STOP` 在开发模式下测过 |

## 15. 一句话总结

```text
在 mission_library.c 里用 ACTION_* 表格里的动作积木拼地图。
先小段测试，再逐步加完整路线。
比赛模式只保留正式 MAP，现场不要误选测试任务。
```
