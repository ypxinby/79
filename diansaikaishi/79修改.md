## 4. 七路循迹传感器模块

相关文件：

```text
track_sensor.c / track_sensor.h
```

### 位映射约定

- raw 寄存器 bit0 ~ bit6 对应 S1 ~ S7
    
- bit0 = 最左侧 S1
    
- bit6 = 最右侧 S7
- bit7 不使用
    
- raw 中 `1` 代表检测到黑线
    

### 引脚与 bit 对应关系

|bit|传感器|引脚|
|---|---|---|
|bit0|S1|B25|
|bit1|S2|B24|
|bit2|S3|B20|
|bit3|S4|A14|
|bit4|S5|B18|
|bit5|S6|B19|
|bit6|S7|B10|

### 黑线电平定义

当前配置：

```c
#define TRACK_BLACK_LEVEL (1)
```

含义：

```text
高电平代表黑线
```

本次 7 路模块测试发现 OLED RAW 黑白方向相反，已在底层做反向适配，软件仍保持：

```text
RAW 中 1 代表黑线
RAW 中 0 代表白底
```

如果重新上板后仍发现白底显示为 1，修改路径：

```text
track_sensor.h line 8
```

将其改为：

```c
#define TRACK_BLACK_LEVEL (0)
```

### 循迹误差权重

```text
S1    S2    S3    S4   S5   S6   S7
-300  -200  -100  0    100  200  300
```

误差逻辑：

```text
黑线偏左：误差为负
黑线居中：误差接近 0
黑线偏右：误差为正
```

## 11. PD 循迹控制核心

相关文件：

```text
car_controller.c / car_controller.h
```

运行数据结构体：

```c
AppRuntime g_appRuntime
```

### g_appRuntime 包含内容

```text
当前圈数
传感器原始值
黑线计数
误差
历史误差
修正量
左右轮速
丢线计数
计圈冷却计时
```

### 完整控制流程

```text
1. 每 20ms 读取七路循迹原始数据、黑线数量、误差值
2. 非 RUNNING 状态：电机停机，速度、修正量清零
3. RUNNING 且无黑线 raw=0：立即停机，丢线计数累加，超过阈值进入 ERROR 报错
4. RUNNING 且检测到黑线：清零丢线计数，计算 PD 修正量，输出左右轮差速驱动电机
```

### PD 修正公式

```c
correction = (track_kp * error + track_kd * (error - last_error)) / track_scale;
```

### 轮速输出公式

```c
left_speed  = base_speed - correction;
right_speed = base_speed + correction;
```

> [!note] 修正方向反向处理  
> 如果循迹修正方向反了，可以交换左右轮加减符号，或者直接对 `correction` 取反。

## 12. 当前工程遗留风险点

1. 循迹黑线电平 `TRACK_BLACK_LEVEL` 未实测验证。
    
2. OLED 默认 I2C 地址为 `0x3C`，部分模块可能为 `0x3D`。
    
3. OLED 硬件必须带 SDA / SCL 上拉电阻。
    
4. TB6612 驱动 STBY 引脚必须拉高，否则电机无输出。
    
5. 当前无计圈逻辑，目标圈数仅可修改，无法自动停车。
    
6. 旧直行控制文件 `straight_control` 仍残留初始化调用，尚未清理。
    
7. RUNNING 状态下，只要传感器识别到黑线，电机会直接驱动，因此首次测试必须架空车轮。
