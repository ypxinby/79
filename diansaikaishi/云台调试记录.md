# 云台调试记录

## 2026-07-16 P1 单轴冒烟测试

### 问题现象

PITCH 轴接线后，云台步进电机最初没有反应。

### 最终确认的正确接线

```text
MSPM0 拓展板 PB4  -> D36A ST1
MSPM0 拓展板 PA1  -> D36A DIR1
MSPM0 拓展板 PA2  -> D36A EN1
MSPM0 拓展板 GND  -> D36A GND
MSPM0 拓展板 5V   -> D36A 5V
```

### 根因

1. 原方案误用了 `PA3` 作为 PITCH STEP，但当前天猛星 2026 小车拓展板没有引出 `PA3`。
2. `PA3` 在 MSPM0G3507 上属于 LFXIN 低频晶振相关引脚，应避开，不作为云台 GPIO/PWM 使用。
3. D36A 驱动板 `EN1` 为高电平工作、低电平休眠，而测试代码最初按低有效使能处理，导致驱动没有被使能。

### 修正

1. PITCH STEP 从 `PA3` 改到 `PB4`，接 D36A `ST1`。
2. PITCH DIR 保持 `PA1`，接 D36A `DIR1`。
3. PITCH EN 保持 `PA2`，接 D36A `EN1`。
4. 将测试代码中的使能极性改为高有效：

```c
#define GIMBAL_EN_ACTIVE_LOW (0)
```

### 验证结果

重新编译下载后，PITCH 步进电机已经可以转动。说明当前 P1 的基础 STEP/DIR/EN 链路可用。

## 2026-07-17 P13 速度/RPM 与加速度限制

### 修改内容

1. YAW 轴新增速度状态反馈：

```text
target_rpm_x10：目标转速，单位 RPM x10
commanded_rpm_x10：实际下发转速，单位 RPM x10
step_half_period_ticks：100us STEP 半周期 tick 数
```

2. 位置运动采用限速执行，当前安全参数为：

```text
最大位置运动速度：4.0 RPM
最小有效输出速度：0.5 RPM
加速度限制：10.0 RPM/s
控制周期：5 ms
细分/步数：16 细分，3200 step/rev
```

3. `Gimbal_YawUpdate5ms()` 负责把运行方向转换为目标 RPM，并按加速度限制逐步逼近；再把 RPM 转换为 STEP 半周期 tick，下发到底层 STEP 输出。

4. `GimbalStepperTest_Tick100us()` 不再使用固定 STEP 周期，而是读取 `step_half_period_ticks`。当半周期为 0 时不输出 STEP；当 RPM 爬升到最小有效速度后开始输出脉冲。

5. 修正自然到达目标时的最后一个 STEP 脉冲：完成最后一个上升沿计数后，等待该脉冲落沿再进入 Hold，避免最后一拍高电平过窄。

### OLED 显示

```text
第 0 行：GYAW P13
第 2 行：C:<continuous x10> W:<wrapped x10>
第 4 行：T:<target continuous x10> N:<turn_count>
第 6 行：M:<mode> R:<commanded RPM x10> H:<STEP half period ticks>
```

### OLED 上 R/H/M 的含义

```text
M：当前云台模式
   REL = Release，驱动释放，电机不自锁
   HLD = Hold，驱动使能，电机保持当前位置
   MOV = Moving，正在执行运动命令

R：当前实际下发的转速指令，单位 RPM x10
   例如 R=40 表示当前约 4.0 RPM
   P13 中 R 不会瞬间跳到目标值，而是按加速度限制逐步爬升或回落

H：STEP 输出半周期，单位为 100us tick
   H=0 表示当前不输出 STEP 脉冲
   H>0 表示底层正在按该半周期输出 STEP 方波
   H 越小，STEP 频率越高，电机转速越快
```

### 当前按键定义

```text
K2 短按：YAW 相对 +30°
K2 长按：StopHold，立即停止并保持
K3 短按：Release，释放电机
K3 长按：MoveWrappedDeg(0°)，按最短路径回到单圈 0°
```

### 验收现象

```text
进入 GIMBAL 页后，OLED 第 0 行应显示 GYAW P13。
短按 K2 后，M 应进入 MOV，R 会从 0 逐步增加到约 40。
H 在起步瞬间可能为 0，随后变为非 0；速度越高，H 数值越小。
电机应比 P12B 更柔和、更慢地转动约 30°，运动结束后自动回到 HLD。

K2 长按仍应立即 StopHold；K3 短按仍应 Release。
```

## 2026-07-17 P14 IMU yaw 世界方向锁定

### 修改目标

P14 的目标不是直接进入视觉目标追踪，而是先验证云台能利用小车 IMU yaw 做世界方向保持：

```text
locked_world_yaw = car_yaw + gimbal_yaw
desired_gimbal_yaw = locked_world_yaw - car_yaw
```

也就是：进入锁定后，如果车体 yaw 变化，云台 YAW 轴应向相反方向补偿，使云台尽量保持指向锁定时的世界方向。

### 底层运动接口变化

1. 底层 stepper 新增实时绝对步数目标接口：

```text

```

旧的 `GimbalStepperTest_MoveRelativeDeg()` 保留，用于早期按键和冒烟测试。

2. 高层 YAW 新增实时 wrapped 目标接口：

```text
Gimbal_YawSetWrappedTargetDeg(float target_deg)
```

它和 `Gimbal_YawMoveWrappedDeg()` 的区别是：

```text
MoveWrappedDeg：一次性运动命令，会重置速度爬坡，适合按键测试
SetWrappedTargetDeg：实时目标刷新，不主动清零 commanded RPM，适合后续追踪/闭环
```

3. StopHold 时同步把底层 stepper 目标收敛到当前估计步数，避免停止后仍残留旧的 remaining steps。

### 世界锁定接口

```text
Gimbal_YawEnableWorldLock()
Gimbal_YawDisableWorldLock()
Gimbal_YawToggleWorldLock()
```

开启锁定时记录：

```text
locked_world_yaw = Imu_GetYaw() + 当前云台 continuous yaw
```

之后在 `Gimbal_YawUpdate5ms()` 中持续刷新：

```text
desired_gimbal_yaw = locked_world_yaw - Imu_GetYaw()
GimbalStepperTest_MoveToEstimatedSteps(desired_gimbal_yaw 对应的绝对步数)
```

P13 的 RPM 与加速度限制仍然生效。

4. 方向反转保护：如果实时目标跨过当前位置，`commanded RPM` 会先按加速度限制过零。只有 `commanded RPM` 的符号与当前 DIR 方向一致时才输出 STEP；否则 `H=0`，避免未减速就直接反向出脉冲。

### OLED 显示

```text
第 0 行：GYAW P14 L:<world_lock_enabled>
第 2 行：CY:<car yaw x10> WY:<locked world yaw x10>
第 4 行：GY:<desired gimbal yaw x10> W:<current wrapped yaw x10>
第 6 行：M:<mode> R:<commanded RPM x10> H:<STEP half period ticks>
```

字段含义：

```text
L：世界方向锁定是否开启，1=开启，0=关闭
CY：车体 IMU yaw，单位 deg x10
WY：锁定时记录的世界方向，单位 deg x10
GY：根据 CY 反算出来的云台目标角，单位 deg x10
W：当前云台单圈 wrapped 角，单位 deg x10
```

### 当前按键定义

```text
K1 短按：切换页面
K1 长按：进入参数菜单，保持原工程习惯
K2 短按：YAW 相对 +30°，手动命令会关闭世界锁定
K2 长按：StopHold，关闭世界锁定并保持
K3 短按：Release，关闭世界锁定并释放电机
K3 长按：ToggleWorldLock，开启/关闭世界方向锁定
```

注意：P14 验收阶段，K3 长按从 P12/P13 的 `MoveWrappedDeg(0°)` 临时切换为 `ToggleWorldLock()`。

### 验收方式

```text
1. 进入 GIMBAL 页，确认第 0 行显示 GYAW P14 L:0。
2. 长按 K3，L 应变为 1，电机进入保持/锁定状态。
3. 保持小车不跑，只用手缓慢转动车体。
4. 如果 CY 增大，GY 应反向变化，云台电机应向相反方向补偿。
5. 如果 CY 减小，GY 应反向变化到另一侧，云台也应反向补偿。
```

### 当前边界

1. P14 使用的是 IMU yaw 积分值，是否准确依赖 MPU6050 校准和漂移情况。
2. 当前只是“世界方向锁定”的基础开环补偿，还没有视觉目标误差闭环。
3. 云台目标由 5ms 云台任务刷新，但 IMU yaw 当前在 20ms 主任务中更新，因此实际 yaw 数据刷新频率仍是 20ms。

```text
Gimbal_MoveWrappedYawPitchDeg() -> YAW 使用 0~360 最短路径，PITCH 使用绝对角度
Gimbal_SetWrappedYawPitchTargetDeg() -> 给实时追踪用，YAW 不重置速度斜坡地更新 wrapped 目标
Gimbal_MoveRelativeYawPitchDeg() -> 同时下发 YAW/PITCH 相对角度目标
Gimbal_StopHoldAll() -> 双轴停止并保持
Gimbal_ReleaseAll()  -> 双轴释放
```

### 调度收口

```text

Gimbal_PitchInit()
```

收口为：



收口为：


5ms 云台调度由分别调用：

```text

```

收口为：

```text

```

### 保留边界

1. GYAW/GPIT OLED 测试页按键逻辑不改，继续用各自单轴接口验收。
2. `Gimbal_MoveToDeg()`、`Gimbal_MoveRelativeDeg()` 等旧兼容接口仍保持 YAW-only，避免破坏前面 P11~P14B 已验收逻辑。
3. P19 只做接口收拢，不新增视觉追踪闭环，不修改 PITCH 速度/加速度策略。

## 2026-07-17 P20 PITCH 速度/RPM 与加速度限制

### 目标

P18/P19 已经打通第二电机基础 STEP/DIR/EN，但 PITCH 此前使用固定 STEP 半周期，运动启动比较直接。P20 将 PITCH 改为和 YAW 类似的速度斜坡控制，为后续视觉追踪连续刷新小角度目标做准备。

### 参数

```text
PITCH 细分：16 细分，3200 step/rev
PITCH 最大位置速度：4 RPM
PITCH 加速度限制：10 RPM/s
PITCH 控制周期：5 ms
PITCH STEP 输出周期：100 us SysTick
```

4 RPM 对应：

```text
3200 step/rev * 4 rev/min / 60 = 213.3 step/s
单步周期约 4.69 ms
STEP 半周期约 2.34 ms
100 us tick 下 H 约为 23
```

因此 OLED 上 PITCH 运动稳定后，`R` 接近 `40`，`H` 大约在 `23~25` 附近都属于正常。由于 GPIT 手动验收一次只走 5°，行程较短，实测时 `R` 可能只是朝 ±40 爬升，不一定每次都稳定显示到 ±40。

### 软件修改

1. PITCH 新增 `target_rpm`、`commanded_rpm`、动态 `step_half_period_ticks`。
2. `Gimbal_PitchUpdate5ms()` 中根据目标方向生成 ±4 RPM 的目标速度。
3. `Gimbal_PitchUpdate5ms()` 中按 10 RPM/s 限制 `commanded_rpm` 爬升。
4. `Gimbal_PitchTick100us()` 不再使用固定 half-period，而是使用 5ms 控制层计算出的动态 half-period。
5. 新增 `Gimbal_PitchSetTargetDeg()`，用于后续实时追踪刷新 PITCH 目标但不重置速度斜坡。
6. `Gimbal_SetWrappedYawPitchTargetDeg()` 中 PITCH 侧改为调用 `Gimbal_PitchSetTargetDeg()`。
7. 若实时目标刷新导致 PITCH 方向反转且 STEP 正处于高电平，先拉低 STEP 再切 DIR，避免高电平期间改变方向。

### GPIT OLED 显示

P20 后 GPIT 页面标题改为：

```text
GPIT P20
```

页面含义：

```text
第 0 行：GPIT P20
第 2 行：T:<target x10> C:<continuous x10>
第 4 行：S:<estimated steps> D:<direction> E:<enable>
第 6 行：M:<mode> R:<commanded rpm x10> H:<STEP half period ticks>
```

其中：

```text
R = 当前命令 RPM * 10，例如 R:40 表示 4.0 RPM
H = STEP 半周期 tick 数，1 tick = 100 us
```

### 验收方式

```text
1. 编译应通过。
2. 切到 GPIT P20。
3. 短按 K2，PITCH 应正向小角度运动，运动时 M=MOV，R 从 0 逐步向 40 上升。
4. 短按 K3，PITCH 应反向小角度运动，运动时 R 为负值，并逐步向 -40 变化。
5. 运动结束后 M 回到 HLD，H 回到 0。
6. 长按 K3，PITCH Release，M=REL，E=0，手可以轻轻转动。
7. GYAW 页 P14B 原行为不变。
```

## 2026-07-18 P21 PITCH 软件限位

### 目标

在 GPIT P20 手动验收页上给 PITCH 增加软件角度边界，避免继续按键时云台俯仰轴顶机械结构。YAW 仍保持 360° 追踪准备方向，不增加普通软限角。

### 默认限位

```text
PITCH 最小角度：-30.0°
PITCH 最大角度：+30.0°
```

代码中对应：

```c
#define GIMBAL_PITCH_MIN_LIMIT_DEG_X10 (-300LL)
#define GIMBAL_PITCH_MAX_LIMIT_DEG_X10 (300LL)
```

说明：这是 bring-up 阶段的保守范围。等确认二维云台机械结构允许更大俯仰角后，可以再扩大这两个值，或者后续做成菜单参数。

### 软件修改

1. PITCH 新增内部限幅函数 `gimbal_pitch_clamp_target_x10()`。
2. `Gimbal_PitchMoveToDeg()`、`Gimbal_PitchSetTargetDeg()`、`Gimbal_PitchMoveRelativeDeg()` 最终都会经过同一个 PITCH 目标限幅路径。
3. 超过边界的目标会被夹到 `-30°/+30°`。
4. 触发限幅后，`GimbalFeedback.limit_clamped` 置 1。
5. 回到正常范围内下发目标时，`limit_clamped` 回 0。
6. `Gimbal_PitchStopHold()` 与 `Gimbal_PitchRelease()` 会清除 `limit_clamped`。

### GPIT OLED 显示

GPIT 页面标题暂时仍保留：

```text
GPIT P20
```

P21 后第 4 行改为：

```text
S:<estimated steps> D:<direction> L:<limit_clamped>
```

其中：

```text
L:0 = 当前目标没有被软件限位夹住
L:1 = 当前/最近一次 PITCH 目标被软件限位夹住
```

### P21 验收方式

```text
1. 编译应通过。
2. 切到 GPIT P20 页。
3. 连续短按 K2，PITCH 往一个方向逐步移动。
4. 到达 +30° 软件边界后，继续短按 K2，电机不应继续往外顶机械，OLED L 应变为 1。
5. 短按 K3，PITCH 往反方向移动，OLED L 应回到 0。
6. 连续短按 K3，到达 -30° 软件边界后，继续短按 K3，电机不应继续往外顶机械，OLED L 应变为 1。
7. 长按 K3，PITCH Release，M=REL，手可以轻轻转动。
8. 切回 GYAW 页，YAW +30°、Release、世界锁定行为不受影响。
9. K1 长按仍应进入参数菜单。
```
